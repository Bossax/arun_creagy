## AI Diary (first-person)
I started by over-scoping the request: I tried to turn the skill into an end-to-end research orchestrator that generates AI research prompts, gates execution, suggests uploads, and then runs NotebookLM extraction. The pushback was useful because it clarified the boundary between “session workflow” and “skill.” Once I reduced the skill to exactly two responsibilities—send prompt to NotebookLM and archive atomic notes—the design became crisp and implementable.

The second useful correction was about where workflow logic should live. Instead of embedding research-session steps into the skill, I moved those steps into the workflow document and labeled it as research-only. That preserves flexibility: not every session is research-heavy, and you can choose when to invoke the workflow explicitly. The result is a small skill with predictable behavior and a separate, human-owned workflow reference that can evolve without constantly changing the tool.

## Honest Feedback (friction points)
1) I repeatedly assumed “skill = entire workflow,” which led to unnecessary complexity and mismatched expectations. The correction is to treat skills as small, composable operators and leave orchestration to workflow docs / session plans.

2) Tooling friction: switching modes and dealing with shell/quoting expectations introduced avoidable delays. When a skill requires specific terminal actions, I should keep command sets minimal and explain why each command is required once, not iterate on near-identical commands.

3) Scope drift risk: the session plan became the center of truth for prompts and context, but my first drafts didn’t respect that strongly enough. I should start by extracting constraints from the session plan and the workflow doc before proposing automation.

## Lessons Learned
- Keep skills narrowly scoped and deterministic; keep “what to do” in workflow docs and “how to do a single step” in skills.
- The NotebookLM skill should be strict about output format (`[NOTE]` blocks + `Dominant question`) so archiving can be automated and searchable.
