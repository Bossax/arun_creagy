## AI Diary (first-person)

I started from a frustrating place: the system was authenticated, but every attempt to ask NotebookLM a question would die with a hard timeout. The signal that mattered was that the failure pattern was consistent and fast — it looked less like “NotebookLM doesn’t work” and more like “the request window is too short for what the browser needs to do.” Once I treated this as a latency-budget problem (not a content problem), the fix became concrete: increase the automation’s allowed wait time and remove avoidable slowdowns like stealth typing.

What felt especially delicate was that browser automation failures can look identical from the outside (timeout, closed context, not connected), so I tried to anchor each step in observable state: health checks, process lock diagnosis, and then a controlled configuration change. After applying the MCP env overrides and restarting VS Code, the first successful NotebookLM answer was a relief because it validated the hypothesis and restored trust in the workflow. I ended with a compact, decision-ready extraction rather than another long narrative, since the immediate goal was “usable essence,” not completeness.

## Honest Feedback (3 friction points)

1) The NotebookLM MCP `ask_question` failures were initially opaque: multiple root causes can collapse into the same timeout symptom, and the 60s client-side timeout makes this feel like “it’s broken” even when the browser is still progressing.

2) Browser automation state is fragile in daily use. If a manual browser session (or a background process) touches the same profile, failures can cascade (locks, closed contexts). This increases the cognitive overhead of “just run the query.”

3) Content extraction depends heavily on the notebook’s sources. For the 2015 act, the model can return a confident summary, but without seeing the full statute text in the notebook sources, it’s hard to validate specifics beyond the extracted definitions/mechanisms.

## Lessons Learned

- NotebookLM MCP timeouts can be solved as a latency-budget issue: configure longer browser timeouts and remove stealth delays via MCP server env in [`/.roo/mcp.json`](.roo/mcp.json:1), then restart VS Code to apply.
- For Thai forestry law, “forest” can be defined on land tenure (not canopy thresholds), so extra care is needed when mapping legal definitions to inventory categories (kept as a separate axis in the research notes).
