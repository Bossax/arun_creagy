## Detailed Task Breakdown

### 2) Finalize stakeholder invite list and coverage
- [ ] Starting from the current functional clusters in the stakeholder-fit plan, produce a concrete list of **named organizations** and (if possible) **named roles/people**.
- [ ] Check the **coverage rule**:
  - count organizations and confirm 15–20 total.
  - ensure **no functional cluster is empty**; if any is missing, document the risk and mitigation.
- [ ] Build a simple **attendance matrix**:
  - rows: organizations (or roles)
  - columns: sessions (S1–S5) and key breakouts
  - cells: `must`, `should`, or `optional`.

### 3) Draft invitation attachments

Treat the invitation package as one integrated narrative, with clear sections that can be included in email, on letterhead, or as an attachment.

- [ ] **Concept**
  - 2–3 paragraphs explaining:
    - NFCS context and where this workshop fits.
    - CRDB/NCAIF role as an enabling substrate (UIP element), not NFCS host.
    - Why Human Settlement & Security journeys were chosen.
- [ ] **Expected outcomes**
  - Short list referencing the deliverables:
    - NFCS steering decision package (2 pages).
    - NCAIF website blueprint annex.
    - Supporting artifacts (service blueprints, baseline/access tables, governance and MEL outlines).
- [ ] **What participants will gain**
  - Tailored bullets for each cluster type (e.g. economic planners, local governments, private sector, data owners, DRM actors, NFCS governance actors).
  - Show concrete benefits (clarity on roles, baselines they can trust, future investment roadmap, etc.).
- [ ] **Agenda**
  - Name each session (S1–S5) in plain language.
  - Indicate which parts are plenary vs breakout and roughly where each stakeholder type is most active.
- [ ] **Preparation guide**
  - What to read beforehand (selected excerpts from interim, Chapter 2 synthesis, or briefing notes).
  - What data, examples, or decisions participants should bring (e.g. recent events, local case examples, gap lists).
  - Practical notes (language, expected level of detail, whether they will be asked to speak).

### 4) Slide deck outline (for steering and participants)
- [ ] Use the already-prepared NotebookLM prompt (or similar) to generate slide titles and bullet content.
- [ ] Ensure the flow matches the logic development order:
  - bridging NFCS and CRDB
  - constraints (governance, TOR website blueprint)
  - approved narrative choice
  - evidence and stakeholder needs
  - workshop design (sessions, roles)
  - deliverables and how they are built from session outputs.
- [ ] Mark where slides can be reused in the invitation package, internal briefings, and in-room facilitation.
