## AI Diary (150+ words)

I felt the session was mostly about reducing ambiguity. The CRI Phase 2 work sits in a messy reality: the indicators are a mix of binaries, small ordinal ladders, and a few counts and ratios. Without an explicit harmonization rule, the index can become a pile of ad-hoc choices that no one can defend later. I noticed that the Phase 2 methodology already contains the right instincts (composite scoring for binary, two-speed Baseline/Target stance, confidence overlay), but it doesn’t fully specify the mathematical “glue” for cross-unit consistency. Writing the harmonization note helped me translate those instincts into a concrete, reviewable pipeline.

I also felt a tension when mapping “asset indicators” to “asset pillars.” Some indicators are obviously assets (shelters, EWS, green buffers), but others are borderline institutional structures (governance bodies) or human resources (medical teams). I made myself slow down and use the pillar definitions explicitly rather than relying on quick keyword matches. That produced a mapping table that is at least audit-friendly, even where the boundaries are uncomfortable. Overall, I’m more confident now that we have a written artifact that can be revised deliberately instead of patching decisions in people’s heads.

## Honest Feedback (100+ words, 3 friction points)

1) Tooling friction: when asked to “use Perplexity/Brave,” the available automation path was unreliable, which forced a workaround via a prompt file. That creates execution risk because it shifts work to manual copy/paste.

2) Boundary friction (Asset vs Governance): the Phase 2 shortlist mixes concepts that look like institutional capacity (e.g., “governance bodies”) into the Asset dimension. That makes pillar assignment inherently ambiguous unless we decide whether we allow some “institutional stock” to count as an asset, or we force it back into governance/process.

3) Consistency friction: we can say “composite scoring” and “confidence overlay,” but unless we write down the normalization family per indicator (binary / ordinal / count cap / ratio bounds) and the missingness rule, teams will implement inconsistent transforms across pillars. The new harmonization note reduces this, but it still needs to be operationalized in the pipeline.

## Lessons Learned

- A Phase 2 method can mention “composite scoring” and “confidence overlay” but still fail in practice unless it includes explicit harmonization rules for non-binary indicators and explicit missingness handling.
- Manual mapping tables are valuable because they make category-boundary discomfort visible (which is better than hiding it in “automatic” keyword rules).
