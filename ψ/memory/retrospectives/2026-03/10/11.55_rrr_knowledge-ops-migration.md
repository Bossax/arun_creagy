## Honest Feedback (friction points)
1) I struggled with command execution because I relied on multi-line shell constructs and assumed consistent behavior across shells. That is a recurring failure mode in this environment: when a command fails, it can fail *silently* or partially, and the next step compounds the error.

2) I didn’t enforce “verify state after each step” early enough. Several times I moved forward without proving that the destination folders existed and contained the expected files, which forced rework.

3) I mixed responsibilities between “research workflow alignment” and “engineering migration.” The conceptual plan was solid, but I should have immediately converted it into an execution checklist with quick validations (counts + spot-check samples) and locked the execution method (scripts) earlier.

## Lessons Learned
- In hybrid Windows + Git Bash environments, **prefer small scripts over complex inline shell**; it reduces ambiguity and makes the process repeatable.
- Treat file-system refactors like data migrations: **log moves, prevent overwrites, verify counts**, then proceed.
- Your workflow preference is capture-first and organize-later: **inbox triage queues are the correct “front door.”**
