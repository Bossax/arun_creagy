The biggest friction was the installer ambiguity: running a global installer from the repo root silently redirected outputs into the project and wiped out the local skill layout I intended to keep. That made it hard to reason about what was “global” versus “local” and led to accidental deletion of tracked files. Second, I had to keep re‑clarifying the scope and rules (SKILL‑only vs script‑based), which slowed momentum and created unnecessary backtracking. ## Lessons Learned
- Always treat `oracle-skills install` as *scope‑sensitive*; run it from the intended root or it may populate the repo.
- For standup/recap ordering issues in this hybrid environment, `bash -lc` around `ls -t` is the reliable fix.
- Keep local skill overrides minimal and explicit: a small SKILL.md that points to the local script beats duplicating upstream logic.
