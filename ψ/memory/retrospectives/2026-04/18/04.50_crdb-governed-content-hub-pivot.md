## Architecture Decisions
- **Governed Content Hub**: Static pages are treated as "Managed Entities" with registered owners and data source dependencies.
- **Source Registration Protocol**: Official publishing is gated by source registration in the metadata catalog.
- **Decoupled Architecture**: CMS (static content) is decoupled from the Metadata Catalog (governance) and the Data Lake (evidence).

## Lessons Learned
- **Managed Static Entities**: Static web pages and infographics should be modeled as entities with metadata links to their evidence sources. This prevents the "Orphan Knowledge" problem where the logic behind a policy claim is lost once published.
- **Strategic Audit Trails**: Explicitly linking Evidence (E-ID) to Triggers (T-ID) and then to Change Logs (CH-ID) creates a "why-chain" that justifies structural pivots to stakeholders.
