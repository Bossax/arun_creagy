## Architecture Decisions

- The foresight report should be drafted in isolated subsections first, then assembled, not drafted directly in one monolithic file.
- The reader-facing structure must follow the skeleton in the base report file, even when the prose is substantially rewritten.
- Audience-facing text must never mention draft rounds, integration workflows, or invisible artifact mechanics.
- Citation design must distinguish public prose from internal evidence traceability, with canonical source IDs and claim mapping.

## Honest Feedback

Three friction points stood out. First, I overestimated the adequacy of the first integrated report simply because it was complete in file terms. That was a real miss in judgment. The user did not need “a file with content”; they needed “the right content under the right headings with no backstage language.” I should have validated that before presenting it. Second, my integration behaviour drifted toward summarisation in places where restoration was required. This happened especially in Chapters 3 and 4, where I preserved some narrative logic but did not fully restore the structural detail and subheadings the reader needed. That reduced trust. Third, I was not strict enough about the difference between internal workflow notes and public prose. Even after being told clearly that references like “Draft 2” were unacceptable in the audience-facing file, I still needed a dedicated pass to stamp them out. That means I need a stronger internal preflight checklist for public Thai reports: skeleton completeness, audience-facing language only, no invisible process references, and explicit verification against the user’s named reference structure.

## Lessons Learned

- A single-file report is not complete unless every required heading from the agreed skeleton is present and populated.
- For Thai formal reports, integration quality must be judged by reader trust, not by file completeness.
- Never allow assembly, draft-version, or workflow language into audience-facing prose.
- Drafting by subsection is strong, but integration must explicitly restore structural obligations from the base report.
