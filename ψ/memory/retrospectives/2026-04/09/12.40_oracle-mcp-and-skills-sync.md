## Lessons Learned

1. **Run MCP inside the same container as the HTTP engine.** Wiring `oracle-v2` via `docker exec -i oracleworkspace-oracle-arun-creagy-1 bun src/index.ts` ensures MCP and HTTP share the same Bun version, `/vault` mapping, database, and vector index. Splitting runtimes between Windows and Linux would create path and state divergence.
2. **Treat `.roo/mcp.json` as code, not mere config.** A missing comma or an incorrect Docker invocation (`docker run src/index.ts`) can silently prevent MCP servers from starting. MCP config deserves the same level of review and version control discipline as application code.
3. **Oracle skills updates affect agents, not Roo.** Running the oracle-skills CLI installer updates Claude Code and Codex skills in `~/.claude` and `~/.codex`, but does not touch `~/.roo`. Roo’s skills are governed by its own directories and repo content, so “global skills update” must be interpreted in the correct context.
