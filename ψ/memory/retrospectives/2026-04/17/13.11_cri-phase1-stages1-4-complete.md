## Honest Feedback
1. **Tooling Friction**: The default Miniconda environment had broken pyexpat DLLs, which caused significant delays in XML parsing. Using uv solved this, but it required a diagnostic detour.
2. **Path Complexity**: PowerShell's handling of the ψ character in absolute paths requires constant vigilance. I had to manually resolve several pathspec errors.
3. **Data Loss Risk**: I performed a recursive Remove-Item on an untracked directory before ensuring all "invisible" files (like READMEs) were moved. I need to be more surgical with cleanups.

## Lessons Learned
- **Medallion Discipline**: Separating Bronze (Raw) from Gold (Master) allows for safe re-runs of the ETL pipeline when source data changes.
- **uv for Speed**: uv venv and uv pip are orders of magnitude faster than traditional virtualenv tools in a CLI context.
- **Canonical Spines**: Always anchor multi-source data (BMA/DDPM) to a government-standard code (DOPA) rather than Thai text names to avoid fuzzy matching nightmares.
