# In CRDB/NCAIF work, treat CDM governance as an operational contract that prevent

In CRDB/NCAIF work, treat CDM governance as an operational contract that prevents the platform from degenerating into a portal-of-portals. The key deliverable is a domain ownership + boundary model (host/mirror/link/interpret-only) that makes catalog-first and content-as-asset execution possible. Hybrid/half-breed CDMs are expected when integrating existing products and academic domains; the failure mode is leaving ownership/boundary decisions implicit. Align baseline inventory fields to the IT catalog’s enforced metadata schema; do not create a competing metadata universe. Ask decision makers to ratify internal domain ownership + translation rules, not to command external teams.
