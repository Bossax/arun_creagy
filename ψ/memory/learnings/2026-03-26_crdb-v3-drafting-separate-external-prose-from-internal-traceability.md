# Learning: Separate external-facing prose from internal traceability (CRDB Interim Report v3)

When drafting a client/audience-facing report in Thai (e.g., DCCE), do not embed internal scaffolding into the deliverable.

## Pattern
- Produce **two layers**:
  1) **External/DCCE-facing draft**: paragraph-based narrative; minimal bullets; no repo-internal markdown links; no “to verify” blocks.
  2) **Internal traceability pack**: claim/paragraph → evidence artifact mapping + QA/missing-data log.

## Why
- Internal links and working notes increase internal confidence but reduce external readability and credibility.
- Traceability is still necessary (no fabricated claims), but it belongs in an internal appendix.

## Operational rules
- Bullets are allowed only for:
  - Short executive summary
  - True enumerated lists (e.g., List A / List B)
- All “ข้อมูลที่ต้องยืนยัน/ยังขาด” items go into an internal QA log file.
