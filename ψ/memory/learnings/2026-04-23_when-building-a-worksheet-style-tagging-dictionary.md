# When building a worksheet-style tagging dictionary that is meant to mirror an ex

When building a worksheet-style tagging dictionary that is meant to mirror an existing governed comparator, the `evidence` column must follow the same epistemic contract as the comparator. In this CRI asset-indicator session, `CRI_Asset_Tagging_Dictionary_v1.md` initially cited internal synthesis artifacts in the `evidence` column, while the governance-side analogue `CRI_Capacity_Tagging_Dictionary_v3.md` cited literature/report anchors. Internal artifacts can support derivation and audit trail, but they are not acceptable replacements for literature/report evidence in a field that signals source authority. The correct pattern is to use internal notes for traceability while the visible `evidence` cell cites the external literature or report source that actually supports the concept.
