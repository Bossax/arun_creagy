# When travel windows overlap with fixed project deadlines (for example, a CRDB pr

When travel windows overlap with fixed project deadlines (for example, a CRDB progress report due mid-trip), treat travel and remote‑work constraints as first‑class in the execution plan: map immovable dates against the travel period, finish prerequisite work (like FGD2) before departure where possible, and reserve focused drafting time during the trip with realistic assumptions about connectivity and energy.
