# When planning artifacts drift into fragmented naming (for example, engagement-he

When planning artifacts drift into fragmented naming (for example, engagement-heavy labels masking architecture and governance tracks), execution clarity declines even if content quality is high. The reliable recovery pattern is to perform a control-surface migration rather than incremental renaming. Effective sequence: (1) one canonical execution spine artifact as control center, (2) one canonical navigation index as entrypoint, (3) domain operational plans only where implementation decisions diverge materially (CDM, NCAIF service, governance/data standards, MVD/L&D), (4) move directly superseded artifacts to archive paths while preserving traceability, (5) update Deliverable/Trigger/Change/Evidence/plan references together, and (6) validate by scanning for old active-path references. If supersession is not encoded across ledgers in the same session, old mental models persist and teams continue navigating by obsolete artifacts.
