# Learning — Two-speed measurement + confidence overlay for administrative resilience indicators

When using administrative datasets (e.g., LPA) as resilience indicators:

1) **Separate “Baseline proxies” from “Target metrics.”**
   - Baseline = what can be measured now at scale (often binary/compliance artifacts).
   - Target = what you ultimately want (process quality / demonstrated performance), which is usually not available at baseline.

2) **Attach a data-richness/confidence score to baseline proxies.**
   - This prevents stakeholders from treating check-box compliance as performance.
   - It also enables honest comparison: profile + uncertainty, not false precision.

3) **Operationalize the dataset as “structural readiness,” not “operational excellence.”**
   - Pair baseline proxy reporting with a “data investment roadmap” that prioritizes target metrics.

4) **Triangulate incentive-linked datasets.**
   - If the dataset affects budgets/bonuses, assume gaming risk.
   - Use audits, cross-datasets, and spot-checks to validate.
