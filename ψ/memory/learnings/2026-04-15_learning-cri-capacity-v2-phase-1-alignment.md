# Learning — CRI Capacity v2 + Phase 1 Alignment (2026-04-08)

## Context

This learning comes from a session focused on aligning CRI’s management surfaces (plan, hub, schedule) with two active tracks:

- Track B: CRI capacity-tagging v2 + CBI crosswalk and v2_CBI dictionary variant.
- Phase 1: Impact / Fiscal Relief pipeline, including new DDPM Excel extracts awaiting tabulation.

Earlier work had already produced rich artefacts—capacity v2 dictionary, CBI bridge, NotebookLM extraction model, and a detailed Phase 1 data pipeline—but these were not fully reflected in the project plan, hub, or schedule.

## Learning

When CRI work advances in bursts (e.g., a strong Track B session or a detailed Phase 1 pipeline design), it is not enough to update only the immediate artefacts. The project remains cognitively expensive to re-enter unless the **navigation and management surfaces** are updated in lockstep:

- The **plan** must name the current execution focus (e.g., “Track B – capacity-tagging v2 + CBI bridge”) and spell out the next 1–2 concrete tasks for each major track (e.g., Phase 1 DDPM Excel tabulation).
- The **hub** must promote the live packs (capacity v2 + CBI, Phase 1 methodology + pipeline) to first-class anchors, so you can jump directly to them without remembering file names.
- The **schedule** must capture the next calendar-bound actions in plain language (“review v2 concept definitions”, “tabulate DDPM Excel into intake tables”), even if the technical detail lives in the pipeline plan.

Without that alignment, the project drifts into a state where the most current work is encoded only in handoffs and retrospectives, which are difficult to scan quickly. Aligning plan, hub, and schedule with each significant methodological or data milestone makes CRI re-entry fast and reduces the risk of losing momentum between sessions.

## Application

- After any substantial CRI session (capacity v2, CBI integration, Phase 1 pipeline, or governance design), explicitly:
  - Update the **plan** with the current execution focus and 1–3 concrete next steps.
  - Update the **hub** to point to any new canonical artifacts (e.g., new v2 views, key handoffs, or pipeline notes).
  - Update the **schedule** with the next day’s actionable tasks, expressed in human language.
- Treat this triad (plan + hub + schedule) as the minimal package that must be kept in sync with handoffs and retrospectives.
