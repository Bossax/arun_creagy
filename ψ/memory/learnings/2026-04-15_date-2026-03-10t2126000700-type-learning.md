# FGD2 polling: questions must map to design decisions (not sentiment)

In workshops, polling is only useful when each question produces an answer that directly supports (or falsifies) a concrete design choice.

Use a decision-mapping pattern:

- **NCAIF sitemap**: ask for a default entry principle (workflow vs persona vs topics vs hybrid), a forced “30-second first click,” and “one-click shortcuts.”
- **CDM**: ask which subject area must be standardized first and what boundary pairing matches real work (national reporting vs local/budget).
- **MVPs**: force “pick 2 Phase 1 commitments,” and explicitly probe **scope realism** (e.g., MVP-2 manual/batch groundwork vs automated pipeline).
- **Governance**: ask for the minimum publishable gate and decision-rights (who can endorse baselines).

This avoids collecting vague agreement and instead creates a decision record that can be defended and translated into updated artifacts.
