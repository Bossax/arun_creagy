# Lessons Learned: CRI Phase 2 execution checklist session (2026-02-18)

- A structured indicator mapping table (indicator → capacity tier → type risk → process-gap → proxy fields) is a powerful way to surface hidden assumptions and measurement gaps; it should become a standard pattern for future index/indicator work.
- Handoffs and execution checklists work best when they explicitly capture constraints and doubts (e.g., missing timestamps, weak collaboration logs), not only plans; this prevents future sessions from re-discovering the same limitations.
- Workflow utilities like /forward and /rrr deliver most value when anchored to concrete project artifacts (plans, critiques, literature notes) rather than used as generic summaries; tight integration turns reflection into direct input for the next design and analysis steps.
