# Learning — Lock structure early and keep internal QA out of final-facing prose

## Pattern
When a base report exists, treat it as a strict schema (topic order + headings). Lock this structure before drafting any audience-facing prose. For long-form reports, use a chapter-by-chapter workflow with intermediate artifacts to prevent drift and to enable incremental evidence injection.

## Why it matters
If structure is not locked early, the document can silently shift genre (brief vs full report) and become irreconcilable with expectations. Also, final-facing documents lose credibility when internal drafting/process language leaks into the prose.

## Operational rule
1) Use the base report headings as non-negotiable constraints.
2) Split into chapter artifacts plus an integration file.
3) Maintain an internal evidence-risk register (HOLD/GAP) outside the reader-facing report.
4) Keep the report prose paragraph-driven; reserve lists for true enumeration.
