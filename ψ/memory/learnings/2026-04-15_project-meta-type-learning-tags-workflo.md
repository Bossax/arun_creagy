# Learning: Knowledge Organization & Indexing as a First‑Class Workflow Step

## What changed

The CRI project implicitly demonstrated a powerful pattern: AI_output research notes in `sources/` were synthesized into thematic knowledge artifacts in `output/`, tied together by an evidence index, and then reflected in methodology documents. In this session, that pattern was:

- Named explicitly as **Knowledge Organization & Indexing (KO&I)**.
- Formalized as a step in the canonical session workflow in `src/00_Meta/00_WORKFLOW.md`.
- Supported by a dedicated SOP (`src/00_Meta/Knowledge_Organization_and_Indexing_SOP.md`) and flexible methodology guidance (`src/00_Meta/Methodology_Guidance.md`).

## Why it matters

- Treating KO&I as a first‑class step (between Execute and Reflect) makes synthesis and evidence‑linking part of the work, not an optional extra.
- The CRI pattern (AI_output → synthesis → evidence index → methodology) becomes reusable across projects without re‑inventing it each time.
- Methodology docs can stay human, narrative, and project‑specific, while still being grounded in explicit evidence via syntheses and indices.

## How to apply it

- At session start, load `src/00_Meta/00_WORKFLOW.md` and decide how heavy the KO&I pass should be.
- After research‑heavy execution, run a KO&I pass per the SOP:
  - Select AI_output and other relevant notes.
  - Draft at least one synthesis note in the project’s `output/` directory.
  - Update or create an evidence index mapping sources → syntheses → methodology sections.
- When drafting or revising methodology docs, follow `src/00_Meta/Methodology_Guidance.md` and reference the syntheses and evidence index instead of embedding full literature reviews.

Over time, this should make project knowledge more searchable, auditable, and easier to re‑use across sessions and projects.
