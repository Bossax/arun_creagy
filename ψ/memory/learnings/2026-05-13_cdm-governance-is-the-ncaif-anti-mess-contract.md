# Learning — CDM governance is the NCAIF “anti‑mess” contract

Date: 2026-05-13

## Pattern

In CRDB/NCAIF work, **CDM governance** must be treated as an operational contract that prevents the platform from degenerating into a “portal‑of‑portals.” The governance deliverable is not prose; it is a **domain ownership + boundary model** that makes catalog‑first and content‑as‑asset execution possible.

## Why it matters

When the organization has many existing products, reports, and analytical workflows, a CDM will inevitably become a “hybrid/half‑breed” between product realities and academically rigorous domains. This is expected. The failure mode is leaving ownership and boundary decisions implicit.

## Implementation cues

- Produce a **CDM domain ownership + boundary matrix** using a simple boundary vocabulary (e.g., host / mirror / link / interpret‑only).
- Ensure content outputs (pages, articles, infographics) are governed as assets and mapped to CDM subject areas.
- Align baseline inventory metadata fields to the IT catalog’s enforced metadata standard; do not create a competing metadata universe.
- Ask decision makers to ratify only what they can control (internal domain ownership + translation rules), not external teams.
