# Learning: CRI capacity dictionary v2 review and handoff

**Date**: 2026-04-09
**Project**: DCCE_CRI

## Pattern / Learning

1. **Use a dedicated review summary before touching a canonical dictionary.** When refactoring a capacity tagging dictionary, it is safer and clearer to (a) collect comments and context into an inbox note, (b) write a separate review summary with per-concept recommendations and a concept table, and only then (c) apply edits to the canonical dictionary file. This avoids half-migrated states and preserves a human-readable rationale for each change.

2. **Embed system constraints directly into indicator design.** For Thai climate governance, indicators that rely on budget tagging, procurement timelines, or dashboard coverage must acknowledge where control actually sits (for example, central Budget Bureau rules vs municipal discretion). Treat these as either Target-dominant or contextual indicators until protocols mature, instead of assuming they are fully measurable Baseline metrics today.

3. **Pair every deep review session with a handoff and a next-session plan.** Writing both a handoff note in `ψ/inbox/handoff/` and a project-local next-session plan under `ψ/incubate/.../output/` turns abstract review into a concrete execution path. Future sessions can start from the plan, not from re-reading the entire dictionary.
