# When using administrative datasets (e.g., LPA) as resilience indicators, apply a

When using administrative datasets (e.g., LPA) as resilience indicators, apply a two-speed measurement stance: explicitly separate Baseline proxies (measurable now, often binary/compliance) from Target metrics (process quality/performance, usually missing). Attach a data-richness/confidence score (0–3) to baseline proxies to prevent false precision and check-box compliance being interpreted as performance. Communicate outputs as profile + gap report with confidence cues, and include a data investment roadmap for collecting target metrics. For incentive-linked datasets, assume gaming risk and triangulate with audits/cross-datasets/spot-checks.
