# Learning – PDF-first Evidence Discipline for CRDB

## Pattern

When building evidence artefacts for a legal or quasi-legal deliverable (like the CRDB interim report), **the original signed or submitted documents (PDFs) must remain the only canonical sources of text**. Any markdown outlines, structured maps, or “v1 summaries” are tools for navigation and drafting, not sources-of-truth.

In this session, I accidentally let a markdown outline of the interim report behave as if it were canonical. That created confusion about where citations and evidence anchors should point. The correction was to (1) explicitly label the outline as non-canonical, (2) restate in the Evidence Gathering Plan that all evidence must be anchored to the PDFs in `ψ/incubate/DCCE/CRDB/inbox_source`, and (3) rebuild artefacts for TOR 5.2.1–5.2.5 and 5.3.1–5.3.2 using PDF sections/appendices as the reference.

## When to Use

- Any time I am creating tables, notes, or matrices that will be used as “evidence” for a report.
- Whenever a project has both PDFs and markdown/Word/Notion derivatives of the same document.
- Especially for government, donor, or legal deliverables where the submitted PDF is what actually “exists” in the external world.

## How to Apply

1. **Declare canonical sources explicitly** in the plan (e.g. “v1 canonical text = these PDFs in inbox_source”).
2. **Mark helper files as non-canonical** at the top, with a short disclaimer and a pointer back to the canonical PDFs.
3. **Anchor all evidence artefacts to PDFs**, using page/section and appendix labels, even if the analysis work happens in markdown.
4. **Design tasks one-TOR-subsection at a time**, so that each artefact can be checked against both the TOR text and the canonical PDFs.
