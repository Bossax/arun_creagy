# MVPs: one MVP may serve multiple workflow patterns (via composition), but each M

MVPs: one MVP may serve multiple workflow patterns (via composition), but each MVP should have one primary pattern; enforce functional orthogonality by assigning each capability boundary to exactly one MVP and having others consume shared standards instead of duplicating them. This reduces redundancy and scope creep while keeping reuse high.
