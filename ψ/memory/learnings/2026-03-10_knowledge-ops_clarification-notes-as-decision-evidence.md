# Clarification notes should be moved into project inbox active after application

**Rule**: When a clarification note (Q/A) is used to update a project hub/plan, move it from `ψ/inbox/` into the project’s `ψ/incubate/.../inbox/active/`.

## Rationale
- Keeps the plan authoritative while preserving why decisions changed.
- Prevents repeated re-triage of the same clarification file.

## Optional enhancement
Add a brief pointer from the plan to the clarification note (don’t embed editor-specific links).
