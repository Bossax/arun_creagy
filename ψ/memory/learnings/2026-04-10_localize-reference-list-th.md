# Learning: localizing reference lists in Thai reports (APA Adaptation)

**Date**: 2026-04-10
**Context**: Translating a bibliographic list from English to Thai for a formal foresight report while maintaining source traceability.

## The Patterns
1. **Preserve author identifiers**: In formal Thai reports, foreign author names are typically kept in English (Last Name, First Initial) to ensure that the source can be easily located in international databases. Use Thai conjunctions (e.g., "และ" instead of "&") to integrate them into the Thai sentence flow.
2. **Translate titles with context**: The title of the work should be translated into Thai to make the bibliography accessible to a Thai-speaking audience. If the original title is highly technical, consider including it in parentheses after the translation.
3. **Localize metadata labels**:
    - "Retrieved from" -> "สืบค้นจาก"
    - "n.d." -> "ม.ป.ป." (ไม่มีปีที่ปรากฏ)
    - "Article" -> "บทความที่"
    - "Chapter" -> "บทที่"
4. **Consistency in year markers**: While many Thai reports use B.E. (พ.ศ.), academic-leaning foresight reports often stick to A.D. (ค.ศ.) for foreign sources to match the year on the source itself. Check the project style pack for specific overrides.

## Operational Discipline
- **Expansion, Not Replacement**: When asked to "convert" a core documentation artifact, always default to creating a new localized file (e.g., `works-cited-th.md`) rather than overwriting the English original. This honors the "Nothing is Deleted" principle and supports bilingual production.
- **Table formatting**: Maintain the same `Ref ID` or `ART_ID` across language versions to ensure that internal cross-references and claim-evidence maps remain valid.

## Oracle Alignment
- **Nothing is Deleted**: Localizing knowledge means adding a new perspective or "view" of the data, not discarding the source language version.
- **Mechanism-first**: A citation list is a mechanism for building trust. The conversion should sharpen this mechanism for the target audience.
