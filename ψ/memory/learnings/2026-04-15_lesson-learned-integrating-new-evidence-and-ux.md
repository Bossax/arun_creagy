# Lesson Learned — Integrating new evidence and UX/UI principles into NCAIF design

When refining the NCAIF platform, explicitly translate new evidence into design rules: platform realities from Pack A set hard capability boundaries, TOR scope from Pack B defines what can be claimed, and UX/UI principles from Pack C shape navigation depth, information scent, and tool–catalog separation. Document the resulting decisions (what changes, what is Phase 1 guidance only, what is deferred/integrated externally) so the interim report can explain the rationale, not just the new sitemap.
