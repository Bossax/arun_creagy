# Learning — Orientation artifacts must be derivative and wired into canonical navigation

When a repo already has many artifacts, creating “one more index” risks increasing confusion unless it follows two strict constraints:

1) **Derivative only**: the new artifact must contain no new decisions or state — only a minimal purpose statement + links to existing anchors.
2) **Wired immediately**: it must be linked from the canonical navigation surfaces (e.g. Hub/plan) so it becomes a true entry point rather than a hidden file.

This prevents “index sprawl” and keeps trust: the user can ignore the index without losing anything, but can use it to re-orient quickly.

Secondary lesson: semantic ambiguity in filenames (e.g. “meeting decisions” vs “meeting agenda”) must be corrected early because the project-management ledgers depend on clean “planned vs outcome” distinctions.
