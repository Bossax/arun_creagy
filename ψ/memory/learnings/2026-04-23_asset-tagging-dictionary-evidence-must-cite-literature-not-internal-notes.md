# Lesson Learned: Asset tagging dictionary evidence must cite literature, not internal notes

When building a worksheet-style tagging dictionary that is meant to mirror an existing governed comparator, the `evidence` column must follow the same epistemic contract as the comparator. In this CRI asset-indicator session, [`ψ/incubate/DCCE/CRI/output/asset_indicator_dictionary/CRI_Asset_Tagging_Dictionary_v1.md`](ψ/incubate/DCCE/CRI/output/asset_indicator_dictionary/CRI_Asset_Tagging_Dictionary_v1.md) initially cited internal synthesis artifacts in the `evidence` column, while the governance-side analogue [`ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary_v3.md`](ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary_v3.md) cited literature/report anchors. That difference is not cosmetic; it changes the meaning of the worksheet. Internal artifacts can support traceability, but they are not acceptable replacements for literature/report evidence in a field that signals source authority. The correct pattern is to use internal notes for derivation and audit trail, while the visible `evidence` cell cites the external literature or report source that actually supports the concept.
