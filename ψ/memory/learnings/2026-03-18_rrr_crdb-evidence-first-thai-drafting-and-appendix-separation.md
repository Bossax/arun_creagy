# CRDB evidence-first Thai drafting and appendix separation

When building sponsor-facing Thai report sections for CRDB-style work, start by stabilizing the evidence package and chapter division before prose drafting. If the body chapter mixes descriptive landscape material and architecture logic, create a dedicated handoff note and move full technical or structural detail into appendices rather than forcing the main chapter to carry everything.

For Thai report prose, three patterns improved quality noticeably:

1. prefer more natural Thai administrative wording instead of imported English-heavy phrasing;
2. add concrete institutional examples when discussing uncertainty, budgeting, or implementation constraints;
3. insert compact summary tables after dense synthesis sections so the grouping logic is visible to the reader.

For Chapter 1-style architecture writing, explain the CDM through its actual subject areas, sub-domains, and key entities rather than through generic architecture prose alone. For Chapter 2-style landscape writing, keep it descriptive and evidence-led, but group unmet needs and use cases clearly so it does not collapse into agency-by-agency note dumping.

Appendix strategy matters: detailed CDM structure, full use cases, and sitemap drill-down should live in separate appendix sections so the main report can stay readable while still preserving traceability and technical completeness.
