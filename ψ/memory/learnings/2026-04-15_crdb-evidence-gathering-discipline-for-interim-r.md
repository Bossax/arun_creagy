# CRDB Evidence-Gathering Discipline for Interim Report

## Pattern

When drafting complex reports like the CRDB interim, treat **evidence work** as a separate, first-class phase that produces named artefacts per TOR clause and subsection.

1. **Anchor everything in v1**: For each TOR clause, explicitly reference the relevant chapters and appendices of the submitted v1 report (and any inception or governance notes), then treat those as canonical inputs.
2. **Define report-ready artefacts up front**: In the Evidence Gathering Plan, specify concrete outputs (notes, tables, matrices) and "ready-for-writing" criteria before doing any extraction.
3. **Produce artefacts, not prose**: During evidence work, write only intermediate artefacts – no report paragraphs. Each artefact should answer a narrow question (e.g., benchmark table, structure narrative, FGD coverage table).
4. **Keep TOR-scoped boundaries**: Name files and structure content so that each artefact maps cleanly to a TOR clause and target report subsection.
5. **Use the evidence plan as the only task list**: The Evidence Gathering Plan is the source of truth for what to extract next; writing plans should depend on it, not re-invent tasks.

This discipline ensures that when it is time to rewrite the interim report, you are stitching together stable evidence modules rather than thinking and drafting at the same time.
