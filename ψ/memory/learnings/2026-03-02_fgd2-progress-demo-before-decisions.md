# Learning — Run “progress demo + feedback” before forcing FGD decisions

## Pattern

When stakeholder discovery is still in progress, an FGD framed as “finalize decisions” tends to fail (wrong people in the room, debates without evidence, premature commitments). A better intermediate step is an **internal progress demo + feedback** session that:

- builds shared literacy (e.g., CDM as a teaching vehicle),
- surfaces operational pain points,
- collects feedback to improve artifacts,
- and preserves the decision workshop for later.

## Practical rule

- Separate artifacts:
  - Feedback session agenda (activities + feedback capture): [`src/01_Projects/2025-11_DCCE-CRDB/notes/FGD2_agenda_2026-03-02.md`](src/01_Projects/2025-11_DCCE-CRDB/notes/FGD2_agenda_2026-03-02.md:1)
  - Decision blueprint (D1–D11, owners, sign-off): [`src/01_Projects/2025-11_DCCE-CRDB/notes/FGD2_plan_2026-02-26.md`](src/01_Projects/2025-11_DCCE-CRDB/notes/FGD2_plan_2026-02-26.md:1)

## Facilitation messaging

- Say explicitly: “**No decisions finalized today; we collect feedback to improve the pack**.”
- Frame data governance as **data quality + publishing trust**, not bureaucracy.
