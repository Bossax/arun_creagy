# Foresight revision benchmark must test section function, not only content alignment

When revising a complex foresight report, benchmark criteria should not only ask whether a passage is well supported or aligned with team intent. They must also ask whether each section is doing the specific analytical job assigned to it by the report template. A section can be evidence-rich and still fail if it is performing the wrong function — for example, if a system-map section merely repeats narrative prose, or if a baseline-future section does not actually extend structural trends from the previous chapter.

This matters even more when external deep-research outputs are being integrated back into the draft. If prompts return rich material but the editor does not have section-function discipline, the report becomes denser without becoming clearer. The practical fix is to benchmark each revision against two tests simultaneously: (1) team-anchor fidelity, and (2) section-purpose fidelity.
