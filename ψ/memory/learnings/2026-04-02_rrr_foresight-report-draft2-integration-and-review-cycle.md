# Lesson Learned — foresight-report-draft2-integration-and-review-cycle

When drafting a Thai strategic report from many isolated subsection artifacts, the hardest failure mode is not weak prose but **premature integration**. A document can look "complete" while still being wrong in the ways that matter most to the reader: missing required headings, collapsed subsections, lost detail from prior fuller versions, and visible traces of backstage drafting logic. The fix is to make the agreed report skeleton non-negotiable and treat integration as a validation step, not a formatting step. Before presenting an integrated draft, check that every required heading from the base structure exists, that each heading contains real reader-facing content rather than workflow placeholders, and that no text mentions draft rounds, assembly artifacts, or internal process notes. For Thai formal report writing, reader trust depends on invisible craft: complete structure, stable terminology, and zero signs that the audience is seeing the workshop behind the stage.
