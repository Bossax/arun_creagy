# Learning — Workshop polling: avoid “depends” by constraining outputs

When designing Mentimeter (or any rapid polling) for stakeholder workshops, avoid questions where **“it depends”** is the rational, low-risk participant answer (e.g., “Which service level is better?”). Those questions create consensus but **no design guidance**.

Instead, force constrained choices that translate directly into artifacts:

- **Minimum viable output**: table vs dashboard vs map layer vs method note.
- **Minimum trust signal**: source + version + limitations + endorsement.
- **Boundary protection**: where the system must prevent misuse (design, media, budgeting, zoning).
- **Template requirements**: e.g., for briefings, what must be on the **front page** (headline, map, 3 numbers, safe-use box).

This turns polling into **requirements capture** rather than preference capture, and aligns workshop time with tangible next-step design work.
