# Learning: Treat TOR minimum organization counts as coverage thresholds

When a TOR requires “at least N organizations,” interpret it as a **coverage constraint across functional clusters**, not a raw attendance KPI.

**Operational rule**
- Define functional clusters required to make the decision credible (e.g., steering/oversight, coordinating institution, NMHS lead, economic planning, data infrastructure, sensitive data holders, implementers/local, DRM operations, private sector demand).
- Target a planning band (e.g., 15–20 orgs) to absorb drop-offs.
- If any cluster has zero representation, explicitly downgrade what the workshop can claim (inputs to planning, not validated NFCS-ready decisions).

This converts an ambiguous number into a decision-quality guardrail.
