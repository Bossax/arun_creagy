# 1. Use a dedicated review summary before touching a canonical dictionary. When r

1. Use a dedicated review summary before touching a canonical dictionary. When refactoring a capacity tagging dictionary, first collect comments into an inbox note, then produce a separate review summary with per-concept recommendations and a concept table before editing the canonical file.

2. Embed system constraints directly into indicator design. For Thai climate governance, indicators that depend on budget tagging, procurement timelines, or dashboard coverage must acknowledge centralised control and current practice, and may need to be treated as Target-dominant or contextual until protocols mature.

3. Pair every deep review session with a handoff and next-session plan. Writing both a handoff note in ψ/inbox/handoff/ and a next-session plan under ψ/incubate/.../output/ turns conceptual review into an executable path for future sessions.
