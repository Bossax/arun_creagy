# Lesson Learned — chapter structure must be settled before prose drafting

When two adjacent report chapters are tightly coupled, settle their division of labor before drafting either one too far. In this CRDB interim-report session, substantial time was spent restructuring Chapter 1 and Chapter 2 after both had already accumulated content. The strongest improvement came only after making explicit that Chapter 1 should carry the architecture logic and locked NCAIF design choices, while Chapter 2 should carry the current data product landscape, stakeholder data practices, unmet dataset needs, and use-case synthesis. Preserving reference drafts, introducing versioned working copies, and anchoring all planning notes to one canonical writing-plan file made the restructuring manageable. The practical pattern is: anchor file first, chapter division of labor second, analysis artifacts third, prose drafting last.
