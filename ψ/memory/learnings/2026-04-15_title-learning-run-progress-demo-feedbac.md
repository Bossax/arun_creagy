# Learning — Run “progress demo + feedback” before forcing FGD decisions

Learning — Run “progress demo + feedback” before forcing FGD decisions

Pattern: When stakeholder discovery is still ongoing, decision-heavy FGDs tend to fail (wrong decision owners present, debates without evidence, premature commitments). Prefer an internal progress demo + feedback session first to build shared literacy, surface operational pain, and refine artifacts; schedule a later decision workshop to finalize D1–D11.

Practical: separate artifacts for feedback agenda vs decision blueprint; explicitly message “no decisions finalized today”; frame data governance as data quality + publishing trust, not compliance/bureaucracy.
