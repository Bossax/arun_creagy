## B) Chapter-by-chapter rewrite workflow (designed for parallel subtasks)

### B0) Shared setup (must be created before any chapter prose)
Create and treat as shared contracts across all chapter subtasks:
- **Continuity Spine** (one living doc): narrative through-line, key definitions, and cross-chapter promises.
- **Evidence Spine** (claim → source map): one place to check grounding and prevent hallucinated facts.
- **Glossary/Reader Primer**: definitions for readers with no prior knowledge of (1) youth policy, (2) foresight methods, (3) 3 จังหวัดชายแดนภาคใต้ context.
- **HOLD/GAP Register**: a single list of unresolved evidence gaps; each gap has an owner + closure plan.

Suggested file paths (create once; later subtasks append-only):
- `ψ/lab/foresight-report-wrting/artifacts/PM - Continuity Spine (rewrite).md`
- `ψ/lab/foresight-report-wrting/artifacts/PM - Evidence Spine.md` (already exists; extend rather than fork)
- `ψ/lab/foresight-report-wrting/artifacts/PM - Glossary and reader primer (rewrite).md`
- `ψ/lab/foresight-report-wrting/artifacts/PM - HOLD-GAP Register (rewrite).md`

### B1) Per-chapter briefing (the minimum intermediate artifact per chapter)
Before writing any chapter draft, produce a **Chapter Brief** for that chapter. It is the interface between evidence and prose.

Template (one file per chapter; append-only):
1) Chapter purpose (1 paragraph)
2) Reader assumptions + what must be explained from zero
3) Key messages (3–7)
4) Required definitions (terms to introduce + preferred Thai wording)
5) Evidence requirements
   - claims that must be sourced (numbers / legal claims / historical specifics)
   - permissible qualitative synthesis (what can be said without a statistic)
6) Tables/figures required (and where they come from)
7) Cross-chapter dependencies
   - inputs needed from other chapters
   - outputs this chapter promises to later chapters
8) HOLD/GAP items opened by this chapter

Suggested file paths:
- `ψ/lab/foresight-report-wrting/artifacts/brief_ch1.md`
- `ψ/lab/foresight-report-wrting/artifacts/brief_ch2.md`
- `ψ/lab/foresight-report-wrting/artifacts/brief_ch3.md`
- `ψ/lab/foresight-report-wrting/artifacts/brief_ch4.md`
- `ψ/lab/foresight-report-wrting/artifacts/brief_ch5.md`

### B2) Draft loop (per chapter)
Each chapter-writing subtask follows the same gated loop:
1) **Structure lock pass**: create a skeleton that matches Section A headings exactly (no re-ordering).
2) **Reader-first pass**: write transitions and definitions so a new reader can follow.
3) **Grounding pass**: insert sourced specifics; anything unsourced becomes HOLD/GAP (not silent).
4) **Style pass**: align voice and depth to [`ψ/inbox/Foresight Report - writing style reference.md`](ψ/inbox/Foresight%20Report%20-%20writing%20style%20reference.md) and safety rails in [`ψ/memory/resonance/writing-style-th.md`](ψ/memory/resonance/writing-style-th.md)
5) **Continuity pass**: update Continuity Spine; check that promises/definitions match across chapters.

## C) Suggested subtask order for drafting (dependency-aware)

Rationale: readers start with no prior knowledge; chapters must build vocabulary and context before scenarios/strategy.

1) **Shared setup artifacts** (Continuity/Evidence/Glossary/HOLD-GAP)
2) **Chapter 1** (context + purpose + scope + introduce foresight vocabulary)
3) **Chapter 2** (signals + system map + uncertainties that justify scenario logic)
4) **Chapter 3** (baseline + scenarios + desirable future; relies on Chapter 2 tensions)
5) **Chapter 4** (strategy + pathways; relies on Chapter 3 desirable future + scenario stress-test)
6) **Chapter 5** (synthesis; relies on Chapters 1–4)
7) **Front matter**: Thai abstract + English Abstract + keywords (should be written last so it is accurate)
8) **Back matter**: references + appendices packaging (last; depends on final citation decisions)
