
ก็ หน่วยงานแรก หน่วยงานแรกอาจจะขอเป็นทางกรมอุตุฯ ก่อนได้ไหมคะว่า อย่างปัจจุบัน อันนี้ทางดร.ซัน หรือเป็นท่านอื่นในทีมก็ได้นะคะ เพราะเข้าใจว่าหลายๆ ท่านที่มาจากกรมอุตุฯ เนี่ย เราก็อาจจะมีการทำโปรดักต์ที่แตกต่างกันออกไป อย่างตอนนี้ลิสต์มันอาจจะเป็น มีทั้ง มี Thai Weather app, มี TMD พยากรณ์ app, มี Line alert, มีกรมอุตุฯ อุตุนิยมวิทยาเกษตร, มี Climate Field School, มี Automatic Weather Station, มีพยากรณ์อากาศเชิงตัวเลข ก็เยอะมากเลยนะคะ การคาดการณ์ต่างๆ แล้วก็ทางขวามือก็มีพวกรายงานลักษณะของอากาศในแต่ละช่วงเวลาด้วย มีสรุปพายุหมุนเขตร้อน แล้วก็มีดัชนีภูมิอากาศ ตัว Climate Index ด้วย แผนที่อากาศวิเคราะห์ดาวเทียม ไม่แน่ใจว่ามันมี มันมีอะไรที่ถ้าดูเร็วๆ แล้วตกหล่นไปบ้างไหมคะ

ใคร ใครท่านใดจะเป็นคนแชร์คะ มี หรือว่าด้านหลังไหมคะ ช่วยกันแชร์ได้นะคะ ถ้าดูแล้วนี่ส่วนไหนที่มันขาดหาย หรือว่าอะไรที่เป็นตัวไฮไลต์ก็ได้ค่ะ ที่เรารู้สึกว่ามันเป็นไฮไลต์ที่เราทำ แล้วก็มีหน่วยงานอื่นมาขอข้อมูลจากส่วนนี้ พอสมควรเลย แล้วก็กลไกในการแชร์ข้อมูลให้กับหน่วยงานเหล่านี้ มีอะไรบ้างคะ

**[ผู้แทนกรมอุตุนิยมวิทยา]**

คือจริงๆ กรมอุตุฯ เนี่ย ครับ ก็ไม่ได้มีแค่ส่วนพยากรณ์อากาศเนาะ จะมีส่วนที่เป็นภูมิอากาศ ฉะนั้นกรมอุตุฯ นี่ก็ จะมีการให้ข้อมูลในเรื่องของการคาดหมายสภาพภูมิอากาศ แล้วก็เรื่องที่เป็นตัว Automatic Weather Station อันเนี้ย ก็สามารถเข้าถึงข้อมูล แบบไม่ต้องมาขอที่กรมได้เลย เพราะว่าไอ้ส่วนที่มาขอที่กรมเนี่ย เป็นส่วนที่เราใช้คนตรวจ การจัดเก็บก็จะเป็นอีกแบบนึง แต่ถ้าเป็น Automatic เนี่ย ก็สามารถที่จะเข้าในเว็บได้เลย ซึ่งเข้าที่สแกนอยู่ ก็จะมีข้อมูล

_(พิธีกร: 1.6 ใช่ไหมคะ Automatic Weather Station อันนี้เนาะ)_ ทางกรมอุตุฯ เองก็จะมีส่วนของข้อมูลทางด้านดาวเทียม เรดาร์ เนื่องจากผมดูอยู่มีหลายส่วนงานเนาะ เพราะฉะนั้นเค้าก็จะมีเว็บไซต์ที่แยกย่อยในการให้บริการข้อมูล ก็ถ้าดูแล้วก็เหมือนจะ จะเกือบครบ อาจจะขาดไปบ้าง แต่ถ้าอยากจะรู้ว่ามีอะไรบ้าง เข้าไปเว็บไซต์กลางของกรมอุตุฯ เนาะ แล้วมันก็จะมีย่อยๆ ออกไปในส่วนที่ตนรับผิดชอบ อันนี้ก็จะให้ข้อมูล

_(พิธีกร: ทีนี้ถ้าเกิดแชร์มุมมองส่วนใหญ่เราเป็นผู้ให้แล้วกันนะคะ มันมี เรา เราเหมือนมีอุปสรรคอะไรยังไงบ้างว่าเวลาคนจะเข้ามาขอข้อมูลจากเราเงี้ยค่ะ ด้วยระเบียบของเราเอง จริงๆ มันไม่สามารถอยู่ๆ ก็ให้ได้เลย มันต้องมี process ระยะเวลาอะไรยังไงไหมคะ)_ ก็ส่วนใหญ่ที่เข้ามาขอข้อมูล จะเป็นข้อมูลจากสถานีตรวจวัดเนาะ แต่ก่อนเนี่ยก็ต้อง walk-in เข้ามาแล้วก็มาเขียนใบคำขอ หลังๆ เนี่ยก็ เนื่องจากตั้งแต่โควิดนะ สามารถที่จะอีเมลเข้ามาได้ที่ ศูนย์บริการข้อมูลอุตุฯ แล้วก็บอกถึงความต้องการข้อมูล หลังจากนั้นก็จะมีเจ้าหน้าที่เนี่ย จะติดต่อทางเมล์หรือโทร แล้วก็มีการสโคปตัว data ที่จะให้ ก็ถ้าเป็นหน่วยงานราชการ สถาบันการศึกษา ก็จะมีลิมิตที่ให้บริการได้ฟรี แต่ถ้านอกเหนือจากนั้นก็อาจจะมีค่าใช้จ่าย หน่วยงานเอกชนก็เหมือนกันครับ ก็สามารถขอได้ บางทีถ้าขอไม่เยอะเนี่ย ทางกรมก็จะสามารถที่จะจัดให้ได้เลย เหมือนเป็นการบริการ

_(พิธีกร: แล้วปัจจุบันนี้ เหมือนมีกลไกความร่วมมืออะไรไหมคะ ที่กรมอุตุฯ ทำงานกับหน่วยงานภาครัฐอื่นๆ อยู่ค่ะ)_ ก็ จะมีการพูลข้อมูลเนาะ จากตัวคลังน้ำ ตัวคลังน้ำแห่งชาติ ตัวนี้ก็จะเป็นการพูลข้อมูลจากกรมไป เพื่อไปรวมศูนย์ ไม่แน่ใจว่าทาง สทนช. น่าจะเป็นคนทำเป็นศูนย์กลาง แล้วก็ จริงๆ ทางกรมเองก็มี API ให้สามารถลิงก์ข้อมูลโดยตรงได้ด้วย ปัจจุบัน แต่อาจจะย้อนหลังได้ไม่นาน อันนี้ก็อาจจะเป็นอีกช่องทางนึงถ้าหน่วยงานไหนที่อยากได้ข้อมูลที่เป็น data จริงๆ เอาไปใช้งานได้ เป็นส่วนที่เป็นข้อมูลตรวจวัด แล้วก็ข้อมูลพยากรณ์

ส่วนที่ข้อมูลแบบจำลอง NWP เนี่ย เราก็มีบริการที่เป็นข้อมูลที่เป็น ออกมาจากแบบจำลองเลย อันนี้คืออันนึงที่สามารถเข้าถึงได้ ปัญหาอันนึงก็คือ ไอ้ข้อมูลเหล่านี้เนี่ย มันเป็นการให้บริการจากฝ่ายผู้ให้บริการอย่างเดียว ซึ่งมัน ไม่ได้มีฟีดแบ็กจาก user เท่าไหร่ ถ้าเกิดว่าทาง user สามารถที่จะมาฟีดแบ็กได้ว่า เอ๊ะ ข้อมูลเหล่านี้ถ้ามันสามารถเปลี่ยนเป็นลักษณะแบบนี้แบบนั้นได้ ก็อาจจะเป็นประโยชน์ต่อการบริการมากขึ้น ไม่ยังงั้นก็จะ เป็นข้อมูลที่เป็นการให้บริการฝ่ายเดียว

_(พิธีกร: อ๋อ วันนี้ user เยอะเลยค่ะด็อกเตอร์ น่าจะ หมายถึงว่า user ที่เป็นหน่วยงานราชการ ก็ฟีดแบ็กน่าจะ ทางคนอื่นอาจจะเป็น support needs เนาะ ก็น่าจะอยู่ในส่วนนั้น ทีนี้ถ้าเป็น support needs ของกรมอุตุฯ เองอะค่ะ คิดว่าอยาก อยากให้มีอะไรมาช่วยเสริมในการทำงานของเราบ้างอะค่ะ ในเรื่องของตัว Climate Service เนี้ย)_ จริงๆ อยากได้ความ อยากได้ความเข้าใจ user จริงๆ มันต้องมีความท้าทายนะ ไม่นั้นมันก็จะไม่เปลี่ยนอะไร ก็คือ ถ้าเป็นคนทั่วๆ ไปอะ ทำงานแบบนี้ก็สบายอยู่แล้วล่ะ มันก็ไม่ต้องมาเปลี่ยน แต่ถ้ามันมีความท้าทายจาก ความต้องการของกลุ่มผู้ใช้ที่มากขึ้น แล้วก็ลากบอกขึ้น ส่วนเนี้ยสามารถฟีดแบ็กเข้าไปที่กรมได้ กรมจะได้มี action ในการ ปรับเปลี่ยนผลผลิต ซึ่งตอนนี้อย่างกรมอะ มองว่าการทำสิ่งที่เรียกว่า Impact-based forecast เนี่ย มันน่าจะเป็นอะไรที่ตอบโจทย์ของคนหรือผลกระทบในพื้นที่ มากกว่าที่จะให้แค่ปริมาณฝนในพื้นที่กลุ่มใหญ่ๆ เพราะฉะนั้นอันเนี้ย คือสิ่งที่กรมพยายามจะก้าวออกมา สเต็ปแรกในการทำ Impact-based forecast ในเรื่องของการลดความเสี่ยงจากภัยพิบัติ นั่นหมายความว่า ถ้าเกิดกรมพยากรณ์ออกไปว่า จะมีฝนหนักในพื้นที่ ไอ้คำว่าหนักเนี่ย มันเป็นแค่ คำนิยามแล้วก็ใช้ปริมาณในการหาค่า threshold เนาะ แต่เราไม่รู้ว่าไอ้หนักเนี้ยมันแต่ละพื้นที่มันหนัก

_(พิธีกร: บ้านเรามันเป็นหนักบ้านนู้นรึเปล่า)_

ถูกต้อง นั่นคือสิ่งที่กรมพยายามจะก้าวออกมา สเต็ปแรกในเรื่องของการลดความเสี่ยงจากภัยพิบัติ ส่วนสาขาอื่นๆ เนี่ย อาจจะต้องฟีดแบ็กเข้ามาว่า ในภาคเกษตรเราจะต้องเตรียมรับมืออย่างไรถ้าเกิดบอกว่าฝนหนัก กับการปลูกพืช อันนี้ก็จะเป็นอีกมุมนึงที่สามารถฟีดแบ็กเข้ามาได้เลย ผมคิด อันนี้ถือว่าเป็นตัวแทนกรมเลยนะฮะ กรมมีความยินดีอย่างยิ่งที่จะ ที่อยากได้ข้อมูล แม้กระทั่งหน่วยงานรัฐด้วยกัน ผมมองว่าอย่างหน่วยงานรัฐด้วยกันเนี่ย ก็น่าจะทำงานด้วยกันง่ายกว่า เพราะว่ามันเป็นภาพกรอบทำงานที่เป็นการ ไม่ใช่เป็นประชาชนในแต่ละบ้านเรือนเนาะ มันเป็นภาพรวม ซึ่งนำไปกำหนดเป็นนโยบายได้ ในอนาคตด้วย ถ้าเป็นหน่วยงานภาครัฐผมว่าน่าจะ มันเกิดขึ้นได้เยอะ เพราะว่าถ้าเป็นประชาชน บางทีเป็นคำ เรียกว่าอะไร เป็นคำร้องขอแบบที่มันมีรายละเอียดมากๆ เนี่ย มันไม่สามารถทำได้ในเชิงรายละเอียดขนาดนั้น ก็อาจจะเป็นการแชร์ทิศทาง

_(พิธีกร: ขอบคุณทางกรมอุตุฯ นะคะ ก็เป็นหน่วยงานแรกที่เปิดมานะคะ ก็ ฉะนั้นหน่วยงานถัดไปเดี๋ยวเรา เปิดกว้าง เปิดกว้างให้ทุกคนนี่จุกๆ เลยนะคะ แต่ว่าก็เป็น หมายถึงว่า สุดท้ายอะค่ะ ก็ข้อมูลที่ได้มามันจะเป็นข้อมูลที่อยู่ในตัวกรอบการดำเนินงานเรื่องนี้นะคะ ซึ่งทุกคนจะเอาไปใช้ร่วมกันได้นะคะ ไม่ได้เป็นประโยชน์ของหน่วยงานใดหน่วยงานหนึ่งอย่างเดียวนะคะ ถ้างั้นขออนุญาตขยับไปที่ทาง GISTDA ค่ะ ไม่แน่ใจว่า GISTDA ป้ายเราก็เล็กนะคะ ขออภัยขอปรับปรุงรอบหน้า GISTDA นั่งอยู่ตรงไหนนะคะด้านหลัง อ๋อ สักครู่นะคะ)_

**[ผู้แทน GISTDA]**

ครับ สวัสดีทุกท่านนะครับ ก็จาก GISTDA นะครับ ภูรพัฒน์นะครับ ก็ปัจจุบันก็อยู่ฝ่ายความมั่นคงและสังคม ก็ทำงานเชื่อมโยงกับเรื่องข้อมูลของ GISTDA อยู่แล้วนะครับ ก็ในส่วนของ GISTDA เองนะครับ ก็ปัจจุบันเรา เรากำลัง เรากำลังรีดีไซน์เรื่องของโครงสร้างข้อมูลของพวกเราอยู่ นะครับ ก็คือตอนนี้เราจัดกรุ๊ปของข้อมูลในทุกส่วนนะครับ ที่จะโปรวายให้หน่วยงานภายนอกอยู่ ตอนนี้เราก็พยายามจัดทำชุดข้อมูลให้ตรงตาม ในลักษณะของเป็นแต่ละเรื่องนะครับ ในแต่ละเรื่องที่เกี่ยวข้อง ก็ในเรื่อง มีทั้งทางด้านที่เกี่ยวข้องกับเรื่องของภัยพิบัติ ภัยแล้ง น้ำท่วม ไฟป่า ดัชนีความร้อนต่างๆ นะครับ แล้วก็มีในเรื่องของข้อมูลที่เกี่ยวกับเรื่องของการเกษตร เศรษฐกิจต่างๆ นะครับ แล้วก็รวมถึงชุดข้อมูลพื้นฐานต่างๆ รวมถึงด้านสาธารณสุขที่เราเพิ่มเติมขึ้นมา ก็เราก็ได้รับข้อมูลจากทางกรมอุตุฯ นี่แหละครับ มาทำดัชนีบางตัวนะครับ แล้วก็ในชุดข้อมูลต่างๆ ที่เราสามารถวิเคราะห์ได้จากภาพถ่ายดาวเทียม ดัชนีต่างๆ ก็ปัจจุบันเราทำ รวบรวมไว้ แล้วเดี๋ยวจะมี จะมี Open Data ที่เป็นของ GISTDA นะครับ ให้แต่ละหน่วยงานเข้ามาใช้ ทั้งข้อมูลที่เป็น API ต่างๆ หรือว่าข้อมูลที่เป็นระบบต่างๆ ครับ

ก็เดี๋ยว ยังไงจริงๆ อาจจะไม่ต้องลงว่าครบไม่ครบก็ได้นะคะ เพราะว่าเดี๋ยวหลังจากงานนี้ อันนี้แอบบอก เดี๋ยวจะเครียดก่อน เดี๋ยวเราอาจจะมีให้ส่งข้อมูลมาเพิ่มเติมให้กับที่ปรึกษา เรามีช่องทางเป็นตัว SharePoint อะไรเงี้ยค่ะ เป็นคลาวด์ ให้แชร์อีกทีนึงก็ได้ ทีนี้สิ่งที่อยาก อยากให้แชร์เพิ่มเติมอาจจะเป็นเรื่องของ เอิ่ม ข้อเป็น เป็นแก๊ป เป็น needs ละกันนะคะว่า แก๊ปปัจจุบันอย่างเงี้ยค่ะ อย่าง GISTDA เองเนี่ย ถ้าเป็นการของเราการได้มาซึ่งข้อมูลของเรา ก็เป็นระบบของเราอยู่แล้ว ไม่แน่ใจว่ามีปัญหาอะไรไหม หรือว่าถ้าเป็นในแง่ของการ provide ข้อมูลให้กับหน่วยงานอื่นอย่างเงี้ยค่ะ ตอนนี้เรามีเอิ่มอุปสรรคหรือว่ามีช่องทางรูปแบบไหนบ้าง แล้วก็ที่มีอยู่ปัจจุบันเนี้ย มันโอเคแล้วหรือเปล่า หรือว่ามันมีฟีดแบ็กที่คนที่มาขอเราก็ เราได้ยินมาว่ามันลำบากที่จะได้ข้อมูลรึเปล่า อันนี้อาจจะแชร์หน่อยละกันนะคะในมุมมองของ GISTDA ค่ะ

จริงๆ ปัญหาก็น่าจะคล้ายๆ กับที่ๆ เราเจออยู่เป็นประจำก็คือเรื่องของการซิงค์ข้อมูลอะครับ เพราะว่ามันจะมีมาตรฐานของการทำข้อมูลของแต่ละชุดอยู่ตามมาตรฐานต่างๆ เงี้ยครับ ก็มันอาจจะเป็น เราก็พยายามทำให้ตรงตามมาตรฐานที่สุด แล้วก็อย่างที่บอกคือตอนนี้เรารีดีไซน์ตัว Data ทั้งหมดอะครับ ก็จะจัดรูปแบบโครงสร้างข้อมูลตามระบบมาตรฐานชุดข้อมูลที่เราจะมาพับลิกในอีกไม่กี่เดือนข้างหน้านี้อะครับ ก็น่าจะเป็นชุดข้อมูลนึงที่อาจจะไปสนับสนุนแต่ละหน่วยงานในการตามโครงสร้างต่างๆ ได้ดีขึ้นครับ

_(พิธีกร: มีอะไรอยากได้ไหมคะ จากให้เราแบบ มันจะเป็นสิ่งที่เราจะระบุอยู่ในเล่มรายงานเนี้ยค่ะว่า มันเป็น support needs ที่ทาง GISTDA คิดว่าถ้าประเทศมี น่าจะเป็นประโยชน์)_

จริงๆ โดยความคิดเห็นส่วนตัวก็คือเหมือนกับเรื่องของ ด้วยความที่เราทำข้อมูลอย่างข้อมูลภูมิสารสนเทศเนี่ยครับ ทำหลายหน่วยงาน เพราะฉะนั้นก็คือ เราคุยกันมาตลอดเรื่องของการกำหนดมาตรฐานข้อมูลที่ตรงกัน รูปแบบโครงสร้างข้อมูลต่างๆ เพราะว่าบางทีเราทำงานกับข้อมูลชุดนึง แล้วก็ไปทำงานกับข้อมูลอีกชุดนึง มันมีปัญหาเรื่องแก๊ปเรื่องโครงสร้างข้อมูลที่บางทีมันไม่สามารถอาจจะมีการจอยกันได้อะไรเงี้ยครับ ในรูปแบบโครงสร้างข้อมูลที่มันต่างกัน ค่อนข้างลำบาก อันนี้ก็จะเป็นแก๊ปนึงที่คิดว่าน่าจะเป็นประโยชน์แล้วก็หลากหลายหน่วยงานก็น่าจะพบเจอปัญหานี้อยู่ครับ

_(พิธีกร: งั้นเดี๋ยวลองมาดูกันนะคะว่าใครจะเป็นเจ้าภาพงานนี้ เดี๋ยวเล่มนี้ก็คิดว่าน่าจะหาเจ้าภาพ หรือว่าเจ้าภาพร่วมอะไรเงี้ยค่ะ ร่วมกันก็ได้ค่ะ ขอบคุณมากค่ะ)_

_(พิธีกร: งั้นขออนุญาตไปที่หน่วยงานถัดไปเลยนะคะ ทางกรมควบคุมมลพิษค่ะ)_

**[ผู้แทนกรมควบคุมมลพิษ]**

ค่ะ สวัสดีค่ะ ศิรินธิน นะคะ จาก กองจัดการคุณภาพอากาศและเสียง กรมควบคุมมลพิษค่ะ ก็ ในส่วนนี้ของกรมควบคุมมลพิษนะคะ จริงๆ แล้วเราก็เป็นทั้ง Data Provider แล้วก็เป็น Co-producer นะคะ ในส่วนของ Provider เนี่ย เราก็มีการ provide เรื่องของ Air Pollutants นะคะ รวมไปถึงเรื่องของ PM2.5 ที่เป็นการ forecast ล่วงหน้า ซึ่งตรงนี้เองนะคะ เราก็ได้เอาข้อมูลจากทางกรมอุตุฯ นี่แหละค่ะที่เกี่ยวกับพวก Mathematical Model เพื่อที่จะสร้างเรื่องของการพยากรณ์ล่วงหน้า 7 วันให้กับประชาชน

ซึ่งถ้าดูคร่าวๆ แล้วนะคะ ในส่วนแรกก็ถือว่าครบนะคะ เรามีการ provide เรื่องข้อมูลทั้งรายชั่วโมง รายวัน รายเดือน รวมไปถึงรายปี ย้อนหลังไปถึงปี 50 เลยนะคะ แต่ว่าในส่วนของ GD catalog อันนี้อาจจะต้องกลับไปให้ทางกรมเติมข้อมูลเพิ่มนิดนึงเพราะว่าอาจจะยังขาดในหลายๆ ส่วนค่ะ

แล้วก็จริงๆ ในเรื่องของกรอบการดำเนินงานนะคะ เราก็มีทำ MOU เหมือนกันนะคะ กับทาง สทศ. เอง รวมไปถึงกรมอุตุฯ ในการที่จะร่วมแลกเปลี่ยนข้อมูลกันค่ะ แล้วก็มีการทำพวกหนังสือเข้ามาขอ จริงๆ หนังสือที่เข้ามาขอเนี่ย ก็จะมีทั้งหน่วยงาน สถานบันการศึกษา เรื่องของหน่วยงานที่เป็นเอกชน ค่ะ แล้วก็ประชาชนทั่วไปก็มีเข้ามาขอเพื่อที่จะเอาไปทำการศึกษาและก็วิจัยนะคะ รวมถึงคนทั่วไปเองก็สามารถที่จะเข้าถึงข้อมูลต่างๆ ผ่านทางเว็บไซต์ Air4Thai รวมถึง Application Air4Thai ที่ได้เกริ่นไปนะคะว่า เรามีข้อมูลที่ มีให้ค่ะ

แล้วก็ในส่วนของข้อจำกัดใช่มั้ยคะ ต้องเรียนตามตรงว่าด้วยข้อจำกัดก็เรายังมีเรื่องของความต่อเนื่องของข้อมูลอะค่ะ ที่ในบางพารามิเตอร์ บางสถานี อาจจะยังไม่ได้มีความต่อเนื่องเนื่องจากข้อจำกัดด้านงบประมาณ เรื่องของการ maintenance ที่ยังไม่เพียงพอ ที่เราจะกระจายไปกับทางสถานีได้อะค่ะ แต่ว่าในส่วนนี้เองเราก็จะให้ความสำคัญกับเรื่อง PM2.5 เป็นหลัก ซึ่งมันเป็นวาระแห่งชาติหลักของประเทศค่ะ

_(พิธีกร: อันนี้ถามเพิ่มนิดนึงว่า การได้มาซึ่งข้อมูลจากที่บอกว่ามีอนุเคราะห์มาจากกรมอุตุฯ เนี่ย อันนี้เราต้องมีการขอ ปกติเราคือขอเป็นรอบปี หรือว่าเราต้องต้องขอยังไงอะคะ ความถี่ในการขอ)_

ในเชิงเทคนิคอาจจะไม่ได้รู้ขนาดนั้นนะคะ แต่เข้าใจว่ามันจะต้องใช้ตลอด เพราะว่าเรื่อง forecast มันจะต้องรันเข้าสู่ระบบต่อเนื่องอะค่ะ ก็เลยคิดว่าคงจะมีเรื่องของระบบหลังบ้าน API ที่มันเชื่อมโยงกันอยู่แล้ว อ๋อ มีตรงนั้นอยู่ ใช่ค่ะ

_(พิธีกร: ขอบคุณค่ะ ขอบคุณทางกรมควบคุมมลพิษนะคะ ถัดไปเดี๋ยวเราข้ามไปที่หน่วยงานถัดไปเลยนะคะ ทางกรมชลฯ ค่ะ)_

**[ผู้แทนกรมชลประทาน]**

สวัสดีครับ ขออนุญาตแนะนำตัวครับ ผมวาที รัชธร พัฒนาการครับ อดีตผู้อำนวยการศูนย์เฉพาะกิจมาตรการจัดการน้ำ ตกจากภาคใต้นะครับ อดีตครับ

คือข้อมูลของเราเนี่ย ข้อมูลชลประทานเนี่ย ยอมรับว่ามันมีความหลากหลายนะครับ คือชลประทานมันมีคำว่า data นะครับ ปัจจุบันเนี้ยยอมรับว่าจุดอ่อนของเรา data เป็น data จริงๆ มันอาจจะไม่ได้มีการ process ต่อไปเนี่ย นโยบายผู้บริหารของเราคือทำ data ให้เป็น information ให้ได้ ให้ประชาชนเข้าถึงนะครับ อันนี้ก็ฝากว่าถ้าหน่วยงานไหนก็แล้วแต่ รู้ว่ามีข้อมูล data มากๆ ทำให้เป็น information นิดนึง ย้ำเพื่อให้ถึงภาคข้อมูลถึงประชาชน

ภารกิจนะครับ ของเรานะครับ มันจะมีคำว่า Big data หลายหน่วยงานทุกวันจันทร์ของสัปดาห์ จะมีการเรียกประชุมระดับประเทศ เอาหน่วยงานที่เกี่ยวข้องทั้งหมดไปประชุม ถ้าหลายท่านที่เข้าไปร่วมจะได้ทราบดีว่า มันมีการประชุมออนไลน์ด้วยและออฟไลน์ด้วย เพื่อหาบทสรุปของการบริหารจัดการน้ำนะครับ

ก็คืออย่างข้อมูลที่เรานำเสนอก็คือ ปริมาณน้ำในเขื่อนทั่วประเทศ ระดับน้ำในแม่น้ำสายหลัก แล้วก็รายงานฝนสะสม สถานการณ์รายวัน แล้วก็ CCTV แล้วก็การแจ้งเตือนภัยน้ำท่วม แล้วก็ผลพยากรณ์รายวัน กับอีกตัวนึงซึ่งเป็นภารกิจที่เราทำมาร่วมกับ กปน. การประปานครหลวง ก็คือการติดตามความเค็ม กรณีที่มีการขึ้นลงของน้ำทะเล ทำไมมีผลกระทบอย่างไรต่อภารกิจของเรา ก็คือถ้าเกิดว่าน้ำเค็มรุกล้ำ อย่างกรณีของลุ่มแม่น้ำเจ้าพระยา น้ำเค็มรุกล้ำไปเกินสำแล เพราะถ้าเกิดว่าที่สำแลมีการสูบน้ำ มันกระทบหลายหน่วยงานมาก ผู้ใช้น้ำประปา ทั้งอุปโภคบริโภค ภาคอุตสาหกรรม จะได้รับผลกระทบ อย่างงั้นก็คือจะต้องมีวิธีการ ภารกิจคือเฝ้าระวังยังไงว่ากรณีไหน เวลาไหนที่มีน้ำเค็มรุก ก็เราร่วมกับการประปา คือเราจะไปเฝ้าระวัง มีจุดเฝ้าระวัง ถ้าของแม่น้ำเจ้าพระยาจะเฝ้าระวังที่นนทบุรี แต่ถ้าเป็นลุ่มน้ำอื่นก็จะมีจุดเฝ้าระวังแต่ละที่ มีที่ไหนบ้างที่เราต้องเฝ้าระวังเพราะว่ามันจะเกี่ยวข้องกับน้ำกินน้ำใช้ของประชาชน ก็คือแม่น้ำแม่กลอง ท่าจีน เจ้าพระยา และก็แม่น้ำบางปะกง ตรงนี้ต้องมีการเฝ้าระวัง โดยที่ว่าถ้าเกิดปกติน้ำทะเลหนุนทั่วๆ ไป จะต้องมีการเปิดเขื่อนต่อเนื่อง พอรู้ว่าน้ำเค็มรุกเคลื่อนตัวไปถึงไหน จะต้องมีการดึงผล พอไม่งั้นจะเกิดกระบวนการ ถ้าผลกระทบเอาไปผลิตประปา อันนี้เกี่ยวข้องกับหน่วยงานที่นั่งอยู่รอบๆ ผมโดยตรง

กับอีกภารกิจหนึ่ง คือภารกิจที่เกี่ยวกับความจุ ปริมาณน้ำในอ่าง ข้อมูลโทรมาตร ข้อมูลตรวจวัดน้ำฝน แล้วก็ Big data swoc ศูนย์ข้อมูลคุณภาพน้ำ และก็การเพาะปลูก และอีกข้อมูลนึงที่หลายท่านจะไม่ทราบ มันเป็นการร่วมมือระหว่างประเทศ คือกรณีที่มีหน่วยงานที่ อย่างยกตัวอย่าง ชลประทานเราจะมืหน่วยงานทางด้านอุทกวิทยาอยู่ทั่วประเทศ 18 แห่ง แล้วก็จะมีอีกประเภทนึงของศูนย์อุทกวิทยาเราคือสถานีทดลอง ทดลองเรื่องการใช้น้ำของพืชเพื่อใช้น้ำน้อย พวกนี้หลายท่านอาจจะไม่ค่อยได้ยินว่า การแกล้งข้าว ก็คือการทำยังไงให้ใช้น้ำประหยัด และได้ประโยชน์สูงสุด กับอีกอันมาเข้าประเด็นที่บอกว่ามันมีศูนย์อยู่ 8 ศูนย์ ศูนย์อุทกวิทยา 18 แห่ง มันเกี่ยวข้องกับอย่างไร ศูนย์ที่จะมีเกี่ยวข้องกับที่เป็นแม่น้ำที่เป็นชายแดน อย่างสมมติภาคเหนือ เคสที่เราเจอล่าสุดกรณีที่สารปนเปื้อนมันจากแม่น้ำแม่กก ที่เป็นกรณีอยู่ สัตว์ สปีชีส์เนี่ย สัตว์ในแหล่งน้ำเนี่ยมีปัญหา แล้วก็มีผลกระทบต่อประชาชนตอนล่าง ตรงนี้เราต้องอาศัยความร่วมมือระหว่างประเทศเข้ามาเกี่ยวข้อง ล่าสุดมันมีเคสกรณีที่เราดำเนินการ จีนที่เป็นต้นเหตุของปัญหาซึ่งเป็นหลักต้นน้ำ เค้าจะขอตั้งสถานี ก็เลยว่าการตั้งสถานีจะมีฟีดแบ็กต่อบ้านเราอย่างไร ระหว่างตั้งที่เชียงแสน กับตั้งที่สบกก หรือตั้งที่อุบลฯ ตรงนี้ก็เป็นโจทย์ระหว่างหน่วยงานอยู่ที่กำลังช่วยกันหาทางว่า ถ้าให้เค้าตั้งเราได้ประโยชน์อะไร เค้าได้ประโยชน์อะไร

เคสที่สองที่ผมจะยกตัวอย่าง อย่างกรณีศูนย์อุทกวิทยาภาคใต้ มีกรณีการบริหารจัดการลุ่มน้ำโกลก ก็คือ กรณีที่น้ำท่วมเนี่ยมันเป็นการเส้นแบ่งเขตแดนระหว่างประเทศ ในขณะที่เราบริหารจัดการน้ำอีกแบบนึง มาเลเซียบริหารจัดการน้ำอีกแบบนึง กับปัญหาที่กรณีที่ว่าเกิดน้ำท่วม เราจะบริหารจัดการน้ำได้อย่างไร บางครั้งพอมีปัญหาแบบนี้เนี่ย มันก็ต้องมีความร่วมมือ เพราะมันเป็นแม่น้ำระหว่างประเทศ แต่ในขณะที่การแก้ปัญหา ฝั่งเรา ในขณะที่พยายามป้องกันตลิ่งไม่ให้น้ำท่วมกับจังหวัด นราธิวาส ที่เกี่ยวกับแม่น้ำโกลกเนี่ย พอเราไปแก้ปัญหาแบบชิดพรมแดน เราทำหลังคันกั้นน้ำ ในขณะที่มาเลเซีย เค้าไม่ได้ทำแก้ปัญหาแบบชิดพรมแดน เค้าทำมีมีคันขอบตลิ่ง พอถ้าไปทำแนวป้องกัน เข้าไปในพื้นที่ของเค้า และขณะที่ของเราเนี่ย ประเด็นของเรามันจะเกิดกรณีที่ ขอเรียกร้องค่าชดเชย มันอาจจะต้องโยงประเด็น ใครอยู่ในแนวริมตลิ่งก็จะถือว่าได้รับการป้องกัน แต่อยู่นอกแนวตลิ่งก็จะไม่ได้รับการชดเชยประมาณนี้ครับ มันก็ มันจะแตกประเด็น

ภารกิจนึงที่กรมชลฯ เจอ อันนี้มันอาจจะเกี่ยวข้องกับบางเวทีที่ผมไปร่วมมา กรณีลุ่มแม่น้ำบางปะกง เคสที่กรมชลฯ เจอขอค่าชดเชย เคสแบบนี้มันไม่จบง่ายๆ แค่นี้ มันจะขยายต่อไปในอนาคต อย่างกรณีอย่างที่อาจารย์ที่ปรึกษา อย่างกรณีลพบุรี ไม่รู้ว่าอาจารย์จะรู้รึเปล่า ที่บอกว่าลพบุรีรับน้ำแทนชาวกรุงเทพฯ อีหน่อยผมมองเลยว่า คนลพบุรีหรือคนอยุธยาก็ตาม ผมรับภาระแทนชาวกรุงเทพฯ คุณจะชดเชยอะไรให้ผม คนลพบุรีก็ตาม คนอยุธยา นั่นแหละคือสิ่งที่จะเติบโต ล่าสุดมันก็เริ่มขยายไปในพื้นที่อื่น พื้นที่ที่บอกว่าเป็นพื้นที่หน่วงน้ำพื้นที่รับน้ำ อนาคตคือหลายๆ หน่วยงานจะ คือกรมชลฯ จะต้องขอค่าชดเชย จะฝากประเด็นไว้แค่นี้ฮะ

แล้วก็อีกเรื่องนึงก็คือ หลายๆ ครั้งที่ผมทำความร่วมมือกับรัฐกับหลายๆ ประเทศ จากเมื่อก่อนในอดีต เราทำงานร่วมกับ JICA ทำงานกับญี่ปุ่น K-water หน่วยงานที่เป็นของไทยเกาหลี หลายครั้งที่โครงการเหล่านี้เข้ามา ซึ่งในอดีตอาจจะไม่มีหน่วยงานจัดการน้ำ หลายๆ ครั้งโครงการที่เริ่มต้นในอดีตมันมีมาก แต่พอระยะเวลาเปลี่ยนไป โครงการเหล่านี้เครื่องมือเหล่านี้มันไม่ได้รับการต่อยอดประสานต่อ โครงการหลายๆ โครงการก็พับไป ถ้าหากว่ามันมีหน่วยงานกลางบริหารจัดการ ยกตัวอย่างเรากำลังพูดถึงเรื่อง Data Information ถ้าหากว่าโครงการของญี่ปุ่นก็ตาม ของอะไรก็ตาม มันมีหน่วยงานกลางบริหารจัดการหรือพยายามทำให้เป็น ไปในทิศทางเดียวกัน มันก็จะส่งต่อ และล่าสุดเกี่ยวข้องกับอุตุฯ กับหน่วยงานกรม อย่างของญี่ปุ่นเนี่ย ให้โครงการสถานีเรดาร์เรามา แต่ผมก็ต้องให้เครดิตทางอุตุฯ ว่ามีความเชี่ยวชาญเรื่องนี้มากกว่า แต่เครื่องมือเหล่านี้มันไปอยู่ที่หน่วยงานกรมชลประทาน ถ้าหากว่ามีองค์ความรู้ หรือมีองค์ความรู้ทางของอุตุฯ หรือของผมอาจจะด้อยกว่า ทำยังไงให้บูรณาการ ให้ข้อมูลที่สถานีเรดาร์ของกรมชลฯ ที่ได้มาเนี่ย ต่อยอดให้เกิดประโยชน์กับประชาชน ผมว่าคุ้มค่าและเป็นประโยชน์ และญี่ปุ่นไม่ใช่ว่าเค้าจะจบแค่นี้ ถ้าเรามีศักยภาพพอ เอาข้อมูล Database ของสถานีเรดาร์ที่ญี่ปุ่นให้มาเนี่ย เอามาต่อยอด ผมว่าบ้านเราจะได้สถานีเหล่านี้เพิ่ม เป็นการทำความร่วมมือที่ดีครับ ครับ ผมอธิบายแค่นี้ก่อนครับ รอบแรกครับ

**[ผู้แทน สทนช.]**

สวัสดีค่ะ ศุภราณี บุญคำ นะคะ ผู้อำนวยการกลุ่มอุตุนิยมและอุทกวิทยาประยุกต์ นะคะ ศูนย์อำนวยการน้ำแห่งชาติ ค่ะ ภารกิจของ สทนช. นะคะ ก็ตั้งแต่จัดทำแผนแม่บทนะคะ แล้วก็ในเรื่องการกรองโครงการสำคัญที่จะพัฒนาโครงการด้านน้ำนะคะ รวมถึงที่เราจะประเมินความเสี่ยงนะคะ ในภาวะวิกฤติ และภาวะปกติ นะคะ ด้านการบริหารจัดการน้ำ

โดย สทนช. เอง ไม่ได้ provide ในเรื่องสถานีตรวจวัดนะคะ ข้อมูลต่างๆ เนี่ยเราจะใช้รวบรวมจากหน่วยงานที่เกี่ยวข้องนะคะ รวมถึงที่สำคัญก็ข้อมูลจากกรมอุตุฯ นะคะ ก็จะเป็นข้อมูลทาง Observe แล้วก็ข้อมูลคาดการณ์ตั้งแต่ระยะสั้น ระยะกลาง ระยะนานนะคะ ตรงนี้ก็จะใช้ได้ตั้งแต่ในการจัดทำแผนแม่บทนะคะ แผนงานโครงการ แล้วก็บริหารจัดการน้ำ นะคะ

ในส่วนนี้จะขอความอนุเคราะห์ทางกรมอุตุฯ นะคะ ที่จะให้ข้อมูลทาง สทนช. ในการประกอบการบริหารจัดการน้ำ ที่สำคัญก็คือข้อมูลการคาดการณ์ระยะนานนะคะ เป็นการคาดการณ์ล่วงหน้า 6 เดือนนะคะ ตรงนี้อาจจะให้ทาง สทนช. เป็นตัวกลางประสานให้กับทาง กฟผ. หรือ ชป. นะคะ เพื่อไปวางแผนการบริหารจัดการน้ำในเขื่อน นะคะ แล้วก็ในส่วนของดัชนีมาตรฐานน้ำฝน นะคะ ตรงนี้คือค่า SPI เพื่อไปวางแผนการบริหารจัดการน้ำในเรื่องของน้ำท่วมนะคะ แล้วก็อีกส่วนนึงที่สำคัญ ก็ตอนนี้ทางกรมอุตุฯ ทำ IBF นะคะ Impact-based Forecast ตรงนี้ก็จะเป็นตัวสำคัญที่จะให้ สทนช. ไปประเมินพื้นที่เสี่ยง เพื่อเป็นการคาดการณ์แล้วก็ประสานหน่วยงานที่เกี่ยวข้องให้การแจ้งเตือนนะคะ โดยเฉพาะในพื้นที่นะคะ ไม่ว่าจะเป็นเรื่องตัวของภัยแล้งนะคะ ที่ผ่านมานะคะ หลักๆ ก็จะขอความอนุเคราะห์ทางกรมอุตุฯ ประมาณนี้ค่ะ

**[ผู้แทนกรมโยธาธิการและผังเมือง]**

ครับผม ก็ในส่วนของกรมโยธาธิการและผังเมืองนะครับ ภารกิจของกรมก็จะประกอบไปด้วย 4 ด้านด้วยกันนะครับ ก็คือด้านการผังเมือง การพัฒนาเมือง การบริการด้านช่าง แล้วก็การด้านอาคาร ซึ่งในส่วนของตัวโครงการนี้อะครับ เราก็ จะอยู่ในส่วนของ Co-producer ก็คือเป็นผู้ร่วมผลิต ให้ข้อมูลสำหรับการนำไปบริการด้านภูมิอากาศ

แต่ว่าจริงๆ แล้วหน่วยงานของเราน่าจะได้ประโยชน์เป็น user ที่ได้ประโยชน์จากโครงการนี้ เพราะว่า ยกตัวอย่างเช่นในการวางผังเมือง จะต้องมีการพิจารณาข้อมูลในหลายๆ ด้าน ทั้งด้านกายภาพ รวมถึงด้านภูมิอากาศ ก็เป็นส่วนสำคัญในเรื่องของพื้นที่ความเสี่ยงน้ำท่วม ความร้อน ต่างๆ หรือภัยแล้ง

สำหรับผลิตภัณฑ์ที่ได้รวบรวมมา ของกรมเราในเรื่องของผังเมือง ก็จะมีแบ่งเป็น 2 ประเภท ก็คือผังนโยบาย แบ่งเป็น 3 ระดับด้วยกัน ก็คือระดับประเทศ ระดับภาค และระดับจังหวัด แล้วก็ผังเมืองอีกประเภทนึงคือ ผังการใช้ประโยชน์ที่ดิน ก็จะแบ่งเป็น 2 ส่วนด้วยกัน ก็คือผังเมืองรวม อีกอันนึงก็คือผังเมืองเฉพาะ

แล้วก็ในเรื่องของงานบริการด้านการพัฒนาเมือง ก็จะมีพวกเขื่อนกั้น การทำเขื่อนป้องกันตลิ่ง แล้วก็เขื่อนป้องกันน้ำท่วม รวมไปถึงการจัดรูปที่ดินเพื่อพัฒนาพื้นที่

ในส่วนการให้ บริการ ของกรมที่มีก็จะมีระบบที่เรียกว่า DPT Town Square ซึ่งภายในก็จะแยกเป็นบริการต่างๆ 4 ด้าน ที่เราบอก มีการตรวจสอบการใช้ประโยชน์ที่ดิน มีการให้บริการเรื่องแบบมาตรฐาน วัสดุก่อสร้างต่างๆ

สำหรับหัวข้อที่อภิปราย กรอบการดำเนินงานร่วมกับหน่วยงานต่างๆ ที่ผ่านมาในการวางผังเมือง ก็ต้องมีการนำข้อมูลที่เราทำ MOU กับทางกรมอุตุฯ มาใช้ในการวางผัง ไม่ว่าจะเป็นพื้นที่เสี่ยงน้ำท่วม หรือพื้นที่ความเสี่ยงต่างๆ

อย่างการวางผังน้ำ เราก็ใช้ข้อมูลจากทาง สทนช. ด้วย ปัจจุบันเรามีคณะทำงานความร่วมมือระหว่างกรมอุตุฯ ที่เราจะเอาข้อมูลการคาดการณ์ล่วงหน้า เรื่องความเสี่ยงต่างๆ มาใช้ในการวางผัง เพราะว่าปัจจุบันเราวางผังเมือง เวลาที่เราวางผังเมือง เรากำหนดพื้นที่ที่เป็นการพัฒนาเมือง เราจะมองว่าพื้นที่ตรงนี้เป็นพื้นที่เสี่ยงน้ำท่วมหรือเปล่า โดยการดูข้อมูลย้อนหลัง 30 ปี มีปริมาณน้ำฝนน้ำท่าเท่าไหร่ เมื่อมาคำนวณเพื่อวางแผนระบบสาธารณูปโภคของเรา แต่ว่าข้อจำกัดคือ ปัจจุบันด้วยภาวะโลกร้อน ข้อมูลเก่าๆ ที่ดำเนินการมามันไม่เพียงพอ เราต้องคาดการณ์ไปในอนาคต เพื่อทำให้การวางผังเมืองปลอดภัยกับชุมชนมากที่สุด เราต้องการข้อมูลที่คาดการณ์ในอนาคต และข้อมูลในอดีตต่างๆ บางทีขาดหายเป็นช่วงๆ นี่คือข้อจำกัดของเรา

การสนับสนุนที่ต้องการ เราอยากได้ข้อมูลที่เป็นมาตรฐานเดียวกันในทุกหน่วยงาน อย่างเรื่องน้ำ ก็มีหลายหน่วยงานดูแล อยากให้เป็นข้อมูลฐานเดียวกัน และอยากได้ข้อมูลที่ต่อเนื่อง ข้อมูลที่คาดการณ์อนาคต ที่เป็นประโยชน์ต่อการวางผังเมืองในอนาคต 20 ปี

**[ผู้แทนมหาวิทยาลัยธรรมศาสตร์]**

ณัฐชัย จากหน่วยวิจัยอนาคตและนโยบายเมือง มหาวิทยาลัยธรรมศาสตร์ ครับ หน่วยงานของผม ภารกิจหลักก็มี 3 อย่างครับ ก็เป็น co-producer เพราะว่าทางหน่วยวิจัย มีการทำ Risk map ในระดับเมือง ภูเก็ต เราเลือกทำกับวิชิฟ เราใช้ข้อมูลจากดาวเทียม ข้อมูล Climate Index จาก สผ. เองด้วย เพื่อมาทำเป็น Risk map และเป็น user ด้วย ในขณะเดียวกัน ธรรมศาสตร์ก็มีโปรแกรมเทรนนิ่ง ล่าสุดก็มีกับกรมโยธา เทรนนิ่งเรื่องการนำแผนแม่บทไปสู่การปฏิบัติ มีปัญหานึงที่เราเจอเหมือนกัน สถาบันการศึกษา ขอข้อมูลรัฐก็ยากเหมือนกัน มี MOU ก็ตาม บางทีก็ใช้เวลานาน อีกอย่าง เราอยากได้ open data platform ในภูเก็ต มีโครงการทำ data platform เราเป็น partner ช่วยผลักดันร่วมกับจังหวัด และมี proxy ข้อมูล อื่นๆ ด้วย บางทีทำระดับเมือง ข้อมูลบางทีมันไม่ได้ลงสเกลถึง บางทีเราก็ต้องผลิตเอง เช่นทำ survey อะไรแบบนี้ครับ

**[ผู้แทน กทม.]**

จากสำนักสิ่งแวดล้อม กทม. ค่ะ ทาง กทม. เรามีแอปพลิเคชัน AirBKK มีการพยากรณ์ล่วงหน้า มีเรื่องของจุดเสี่ยง มีเรื่องของห้องปลอดฝุ่น แผนที่ห้องปลอดฝุ่น อันนี้ก็เป็นการร่วมมือกับหน่วยงานอื่น มี Line alert ของ สนง.นวัตกรรม ที่ส่งข้อความเตือนภัย และก็ที่เราทำร่วมกับกรมอุตุฯ ก็มีจุดความร้อน ทำนายเรื่องฝุ่นล่วงหน้า ส่วนข้อจำกัด ในแง่การพยากรณ์ บางทีข้อมูลมันอาจจะช้าไปนิดนึง มีดีเลย์ของข้อมูล และก็รูปแบบของข้อมูล บางทีต้องเอามาแปลงเพื่อสื่อสารกับประชาชนให้เข้าใจง่ายขึ้นค่ะ

_(พิธีกร: ทาง กทม. มีการขอข้อมูลจากหน่วยงานข้างนอกบ้างไหมคะ)_ เราไม่ได้ดึงตรงๆ แต่เป็นข้อมูลที่เราทำงานร่วมกับกรมควบคุมมลพิษ แล้วก็กรมอุตุฯ ค่ะ
