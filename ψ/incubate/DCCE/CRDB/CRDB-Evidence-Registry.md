# CRDB Evidence Registry

## Purpose

This registry tracks the main **active, non-archived evidence artifacts** currently used to support CRDB planning, synthesis, reporting, and meeting preparation.

It is intended to answer four practical questions:

1. what evidence already exists,
2. what kind of evidence it is,
3. which gap lens it supports most directly, and
4. where it should feed next.

## Usage and maintenance notes

- Treat this as the **working index** for active CRDB evidence, not as a historical archive.
- Prioritize artifacts that are still being used for current drafting, meeting preparation, or workshop design.
- Do **not** treat material under [`ψ/archive`](ψ/archive/) as active evidence for this registry.
- When a major new synthesis note, meeting note, or stakeholder evidence pack becomes operationally important, add one row rather than rewriting the whole document.
- Use the companion map in [`ψ/incubate/DCCE/CRDB/output/CRDB-Evidence-Coverage-Map.md`](ψ/incubate/DCCE/CRDB/output/CRDB-Evidence-Coverage-Map.md) to interpret overall coverage and remaining information needs.

## Practical reading order
1. Start with [`ψ/incubate/DCCE/CRDB/output/CRDB-Evidence-Coverage-Map.md`](ψ/incubate/DCCE/CRDB/output/CRDB-Evidence-Coverage-Map.md) for the current picture.
2. Use this registry to locate the exact supporting artifacts.
3. Go back to [`ψ/incubate/DCCE/CRDB/Hub.md`](ψ/incubate/DCCE/CRDB/archive/Hub.md) for project-level navigation.
