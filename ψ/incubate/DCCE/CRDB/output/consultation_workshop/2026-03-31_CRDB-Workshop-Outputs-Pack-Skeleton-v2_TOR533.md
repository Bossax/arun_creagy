# CRDB Workshop Outputs Pack Skeleton v2 (TOR 5.3.3 — DCCE/CRDB TOR-facing)

**Purpose**: Make post-workshop drafting **mechanical** by providing TOR-facing headings and templates that map directly to workshop artifacts.

**Scope note**

- This pack is written to evidence **TOR 5.3.3** outcomes: exchange of **datasets**, **information products**, and **needs/demand**, and to feed downstream TOR deliverables (inventories and gap analysis).
- **NFCS** may appear as **background ecosystem context** only. It is **not** the headline framing and not the promised workshop output.

## J) Afternoon annex — NCAIF + data management feedback capture (lightweight)

**Purpose**

- Provide a **low-demand** capture surface for the afternoon segment: DCCE briefing + structured feedback.
- Make the morning-to-afternoon connection **mechanical** by tying feedback prompts to the morning’s bridge artifacts.

### J1. Bridge Board template (morning synthesis → afternoon prompts)

_Assemble this board after Session 3 using photos/transcriptions of room artifacts._

**Panel 1 — Exchange snapshot (from Session 2 sheets)**

- Top exchanged datasets (with access posture + key constraints)
- Top exchanged information products

**Panel 2 — Blueprint highlights (from Session 3 canvases)**

- Key user touchpoints (what the user needs to do)
- Key dependency chains (dataset → product → decision touchpoint)

**Panel 3 — Priority gap themes (from Session 2+3 synthesis)**

- What blocks usability/adoption: access, metadata, quality, timeliness, coordination

**Panel 4 — Feedback prompts (participants respond in the afternoon)**

- “Most realistic to operationalize in 3–6 months (and why)”
- “Key constraints that will block adoption (and where they show up)”
- “Minimum rails / operating touchpoints needed”
