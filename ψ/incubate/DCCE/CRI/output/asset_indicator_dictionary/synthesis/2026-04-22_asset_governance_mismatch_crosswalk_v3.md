## Crosswalk method (practical)

1. Use grouped asset clusters (not concept collapse) to keep the decision surface readable while preserving lineage.
2. Preserve provenance using register ID sets (`AIR-###`).
3. Map each cluster to the nearest v3 pillar/concept, then classify mismatch as:
   - `missing`: governance/process logic not represented in current v3 concept set.
   - `weak`: concept exists but is placeholder-level, partially evidenced, or not yet operationalized in worksheet selection fields.
   - `activation-dependent`: governance logic exists but requires bundled cross-pillar activation not yet encoded as a measurable process chain.
