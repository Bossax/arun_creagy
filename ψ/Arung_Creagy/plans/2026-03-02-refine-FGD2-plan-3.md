- Produce a clear **FGD2 agenda document** (timeboxes + feedback capture).

## Scope + constraints

- Align to continuity and decision framing already defined in the comms strategy ([`src/01_Projects/2025-11_DCCE-CRDB/output/Communication strategy - Selling NCAIF and CDM (FGD2 internal).md`](src/01_Projects/2025-11_DCCE-CRDB/output/Communication%20strategy%20-%20Selling%20NCAIF%20and%20CDM%20(FGD2%20internal).md:1)).
- Pre-read pack will be decided **after** activity set is finalized.

## Revised activity ideas (improved for clarity + feasibility)

### A1. Putting-data-in-a-box (CDM activity)

**Purpose**: Make CDM *visible* as a shared structure for messy datasets, using **Mentimeter quiz format** (no physical materials).

**Format** (20–25 mins total):
- **Setup (5 mins):** Brief CDM explanation + Mentimeter quiz setup with 6–10 dataset examples.
- **Quiz (10 mins):** Participants classify datasets into 3–5 **CDM domains**. Include commonly confused items (flood risk vs flood hazard vs historical hazard).
- **Pipeline link (5 mins):** Show a simplified “from source → CDM → NCAIF product” flow. Use 1 example end-to-end.
- **Key message (5 mins):** CDM enables **data quality, comparability, and accountability**.

**Risks + mitigations**:
- Too technical → keep domain labels simple (People, Place, Event, Asset, Governance).
- Too abstract → use datasets the attendees recognize.

**Output**: A quick tally of the **priority entities** surfaced by participants.

### A2. Internal use-case walkthrough (NCAIF is not a mock-up)

**Purpose**: Make NCAIF “real” by walking through **internal DCCE operational use cases**.

**Format** (15–20 mins):
- Use case 1: **Publishing workflow** (draft → review → publish) and where QA/QC fits.
- Use case 2: **Climate risk communication tiers** (policy / practitioner / public) and how outputs differ.
- Use case 3: **Official dataset search** (finding authoritative sources + metadata).
- Show **where CDM supports the journey** and how tiering reduces usability gaps.

**Feedback tie-in**: collect feedback on which internal use cases are most urgent and where gaps remain.

### A3. Governance as data quality (phasing plan presentation)

**Purpose**: Frame governance as **data quality enablement**, not bureaucracy.

**Format** (10–15 mins):
- Present the **governance phasing plan** (what is required to publish safely in Phase 1).
- Emphasize how governance ensures **quality, comparability, and accountability**.

**Feedback tie-in**: collect reactions to the phasing plan and missing quality gates.

## Where uncertainty remains (requires second opinions)

- **Which internal use cases** should dominate A2 as interviews expand.
- **Feedback method**: live feedback vs post-session survey for prioritization.
- **Time fit**: A1 + A2 live, A3 presentation only.

## Prompt me to decide direction (questions for you)

1. Are the A2 internal use cases still the right ones as new interviews arrive?
2. Should feedback be captured live or via a short follow-up survey?

## Proposed agenda doc structure (draft)

1. **Opening + continuity bridge** (15–20 mins)
2. **A1: Putting-data-in-a-box (Mentimeter + CDM explanation)** (20–25 mins)
3. **A2: Internal use-case walkthrough** (20–25 mins)
4. **A3: Governance phasing plan (data quality framing)** (10–15 mins)
5. **Feedback capture + close** (15–20 mins)
